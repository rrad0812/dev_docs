
# 11.3 Pisanje test slučajeva

[Sadržaj](_00.0-sr.md)

Tokom razvoja, veoma važan korak je testiranje našeg koda kako bismo osigurali njegov kvalitet i integritet. Moramo se uveriti da svaka funkcija vraća očekivani rezultat i da naš kod radi optimalno. Već znamo da je fokus jediničnih testova pronalaženje logičkih grešaka u dizajnu ili implementaciji programa. Oni se koriste za rano otkrivanje i otkrivanje problema u kodu kako bismo ih lakše mogli popraviti, pre nego što izmaknu kontroli. Takođe znamo da se testovi performansi sprovode radi optimizacije našeg koda kako bi bio stabilan pod opterećenjem i mogao da održi visok nivo konkurentnosti. U ovom odeljku ćemo pogledati neka često postavljana pitanja o tome kako se jedinični i performansni testovi implementiraju u Go jeziku.

Jezik Go dolazi sa laganim okvirom za testiranje pod nazivom `testing`, i možemo koristiti `go test` komandu za izvršavanje jediničnih i performansnih testova. Go-ov `testing` okvir radi slično okvirima za testiranje u drugim jezicima. Pomoću njih možete razviti sve vrste testnih paketa, koji mogu uključivati testove za jedinične testove, benčmarking, stres testove itd. Hajde da naučimo o testiranju u Go-u, korak po korak.

## Kako pisati test slučajeve

Pošto se `go test` komanda može izvršiti samo u direktorijumu koji sadrži sve odgovarajuće datoteke, kreiraćemo novi direktorijum projekta `testing` tako da sav naš kod i testni kod budu u istom direktorijumu.

Hajde da napravimo dve datoteke u direktorijumu pod nazivom "gotest.go" i "gotest_test.go"

"gotest.go": Ova datoteka deklariše ime našeg paketa i ima funkciju koja vrši operaciju deljenja:

```go
package gotest
 
import (
    "errors“
)
 
func Division(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("Delilac ne može biti 0")
    }
    return a / b, nil
}
```

"gotest_test.go": Ovo je naša datoteka za jedinično testiranje. Imajte na umu sledeće principe za test datoteke:

- Imena datoteka moraju da se završavaju na "_test.go" kako bi `go test` mogao pronaći i izvršiti odgovarajući kod
- Morate da uvezete `testing` paket
- Sve funkcije test slučajeva počinju sa `Test`
- Test slučajevi prate redosled izvornog koda
- Test funkcije oblika `TestXxx()` uzimaju `testing.T` argument; ovaj tip možemo koristiti za beleženje grešaka ili za dobijanje statusa testiranja
- U funkcijama oblika `func TestXxx(t * testing.T)`, `Xxx` odeljak može biti bilo koja alfanumerička kombinacija, ali prvo slovo ne može biti malo slovo [az]. Na primer, `Testintdiv` bi bio nevažeći naziv funkcije.
- Pozivanjem jedne od metoda `Error`, `Errorf`, `FailNow`, `Fatal` ili `FatalIf` od naših funkcija za testiranje, možemo da ne prođemo test. Pored toga, možemo pozvati metodu `testing.TLog` da bismo zabeležili informacije u dnevniku grešaka.

Evo našeg test koda:

```go
package gotest

import (
    "testing"
)

func Test_Division_1(t *testing.T) {
    // try a unit test on function
    if i, e := Division(6, 2); i != 3 || e != nil { 
        // If it is not as expected, then the test has failed 
        t.Error("division function tests do not pass ") 
    } else {
        // record the expected information
        t.Log("first test passed ") 
    }
}

func Test_Division_2(t *testing.T) {
    t.Error("just does not pass")
}
```

Prilikom izvršavanja `go test` u direktorijumu projekta, prikazaće se sledeće informacije:

```sh
--- FAIL: Test_Division_2 (0.00 seconds)
gotest_test.go: 16: is not passed
FAIL
exit status 1
FAIL gotest 0.013s
```

Iz ovog rezultata možemo videti da druga test funkcija ne prolazi jer smo pisali u ćorsokaku koristeći `t.Error`. Ali šta je sa performansama naše prve test funkcije? Podrazumevano, izvršavanje `go test` ne prikazuje rezultate testa. Potrebno je da navedemo detaljan argument `-v` sa `go test -v` da bismo prikazali sledeći izlaz:

```sh
=== RUN Test_Division_1
--- PASS: Test_Division_1 (0.00 seconds)
gotest_test.go: 11: first test passed
=== RUN Test_Division_2
--- FAIL: Test_Division_2 (0.00 seconds)
gotest_test.go: 16: is not passed
FAIL
exit status 1
FAIL gotest 0.012s
```

Gore navedeni izlaz detaljno prikazuje rezultate našeg testa. Vidimo da test funkcija 1 `Test_Division_1` prolazi, a test funkcija 2 `Test_Division_2` ne prolazi, što na kraju zaključuje da naš skup testova ne prolazi. Zatim, modifikujemo test funkciju 2 sledećim kodom:

```go
func Test_Division_2(t *testing.T) {
    // try a unit test on function
    if _, e := Division(6, 0); e == nil { 
        // If it is not as expected, then the error
        t.Error("Division did not work as expected.") 
    } else {
        // record some of the information you expect to record
        t.Log("one test passed.", e) 
    }
}
```

Ponovo izvršavamo `go test -v`. Sledeće informacije bi sada trebalo da budu prikazane - test paket je prošao ~:

```sh
=== RUN Test_Division_1
--- PASS: Test_Division_1 (0.00 seconds)
gotest_test.go: 11: first test passed
=== RUN Test_Division_2
--- PASS: Test_Division_2 (0.00 seconds)
gotest_test.go: 20: one test passed. divisor can not be 0
PASS
ok gotest 0.013s
```

## Kako napisati testove stresa

Testiranje opterećenja se koristi za otkrivanje performansi funkcija i ima neke sličnosti sa jediničnim testiranjem (kojim se ovde nećemo baviti), međutim, moramo obratiti pažnju na sledeće tačke:

- Testovi opterećenja moraju pratiti sledeći format, gde XXX može biti bilo koja alfanumerička kombinacija, a prvo slovo ne može biti malo slovo.

```go
funkcija BenčmarkXXX (b *testiranje.B){…}
```

Podrazumevano, Go test ne izvodi testove opterećenja funkcionalnosti. Ako želite da izvršite testove opterećenja, potrebno je da postavite zastavicu `-test.bench` u formatu: `-test.bench="test_name_regex"`. Na primer, da biste pokrenuli sve testove opterećenja u vašem paketu, pokrenuli biste `go test -test.bench=".*"`.

U vašim testovima opterećenja, ne zaboravite da koristite `testing.BN` za bilo koja tela petlji, kako bi se testovi mogli pravilno pokrenuti.

Kao i ranije, imena testnih datoteka moraju se završavati sa `_test.go`.

Ovde kreiramo datoteku za test opterećenja pod nazivom "webbench_test.go":

```go
package gotest

import (
    "testing"
)

func Benchmark_Division(b *testing.B) {
    for i := 0; i < b.N; i++ { // use b.N for looping
        Division(4, 5)
    }
}

func Benchmark_TimeConsumingFunction(b *testing.B) {
    b.StopTimer() // call the function to stop the stress test time count

    // Do some initialization work, such as reading file data, database connections and the like,
    // So that our benchmarks reflect the performance of the function itself

    b.StartTimer() // re-start time
    for i := 0; i < b.N; i++ {
        Division(4, 5)
    }
}
```

Zatim izvršavamo `go test -file webbench_test.go -test.bench =".*"` komandu, koja daje sledeće rezultate:

```sh
PASS
Benchmark_Division 500000000 7.76 ns / op
Benchmark_TimeConsumingFunction 500000000 7.80 ns / op
ok gotest 9.364s
```

Gore navedeni rezultati pokazuju da nismo izvršili nijednu od naših TestXXX jediničnih test funkcija, već smo umesto toga izvršili samo naše BenchmarkXXX testove (što je tačno onako kako se i očekivalo). Prvi Benchmark_Division test pokazuje da je naša Division() funkcija izvršena 500 miliona puta, sa prosečnim vremenom izvršavanja od 7,76 ns. Drugi Benchmark_TimeConsumingFunction pokazuje da je naša TimeConsumingFunction funkcija izvršena 500 miliona puta, sa prosečnim vremenom izvršavanja od 7,80 ns. Konačno, prikazuje ukupno vreme izvršavanja našeg test paketa.

## Rezime

Iz našeg kratkog susreta sa jediničnim i stres testovima u Go-u, možemo videti da je `testing` paket veoma lagan, a opet prepun korisnih alata. Videli smo da pisanje jediničnih i stres testova može biti veoma jednostavno, a njihovo pokretanje može biti još lakše pomoću ugrađene Go `go test` komande. Svaki put kada izmenimo naš kod, možemo jednostavno pokrenuti `go test` da bismo započeli regresiono testiranje.
