
# 7.2 JSON

[Sadržaj](_00.0-sr.md)

JSON (JavaScript Object Notation) je lagan jezik za razmenu podataka koji se zasniva na tekstualnom opisu. Njegove prednosti uključuju to što je samoopisan, lako se razume itd. Iako je podskup JavaScript-a, JSON koristi drugačiji tekstualni format, što ga čini nezavisnim jezikom. JSON je sličan jezicima C porodice.

Najveća razlika između JSON-a i XML-a je u tome što je XML kompletan jezik za označavanje, dok JSON nije. JSON je manji i brži od XML-a, stoga ga je mnogo lakše i brže analizirati u pregledačima, što je jedan od razloga zašto mnoge otvorene platforme biraju da koriste JSON kao svoj jezik interfejsa za razmenu podataka.

Pošto JSON postaje sve važniji u veb razvoju, hajde da pogledamo nivo podrške koju Go ima za JSON. Videćete da Go-ova standardna biblioteka ima veoma dobru podršku za kodiranje i dekodiranje JSON-a.

Ovde koristimo JSON da predstavimo primer iz prethodnog odeljka:

```json
{"servers":[{"serverName":"Shanghai_VPN","serverIP":"127.0.0.1"},{"serverName":"Beijing_VPN","serverIP":"127.0.0.2"}]}
```

Ostatak ovog odeljka će koristiti ove JSON podatke za uvođenje JSON koncepata u Go.

## Raščlanjivanje JSON-a

### Raščlanjivanje u strukturu

Pretpostavimo da imamo JSON u gornjem primeru. Kako možemo da analiziramo ove podatke i mapiramo ih u strukturu u Go-u? Go pruža sledeću funkciju upravo za tu svrhu:

```go
func Unmarshal(data []byte, v interface{}) error
```

Ovu funkciju možemo koristiti ovako:

```go
package main

import (
    "encoding/json"
    "fmt"
)

type Server struct {
    ServerName string
    ServerIP   string
}

type Serverslice struct {
    Servers []Server
}

func main() {
    var s Serverslice
    str := `{"servers":[{"serverName":"Shanghai_VPN","serverIP":"127.0.0.1"},{"serverName":"Beijing_VPN","serverIP":"127.0.0.2"}]}`
    json.Unmarshal([]byte(str), &s)
    fmt.Println(s)
}
```

U gornjem primeru, definisali smo odgovarajuće strukture u Go-u za naš JSON, koristeći slice za niz JSON objekata i naziv polja kao naše JSON ključeve. Ali kako Go zna koji JSON objekat odgovara kojoj određenoj strukturi? Pretpostavimo da imamo ključ koji se zove Foo u JSON-u. Kako da pronađemo njegovo odgovarajuće polje?

- Prvo, Go pokušava da pronađe (velikim slovima) izvezeno polje čija oznaka sadrži Foo.
- Ako se ne može pronaći podudaranje, potražite polje čije je ime Foo.
- Ako i dalje nema podudaranja, potražite nešto poput FOO ili FoO, zanemarujući razliku između velikih i malih slova.

Možda ste primetili da sva polja koja će biti dodeljena treba da budu izvezena, a Go dodeljuje samo polja koja se mogu pronaći, ignorišući sva ostala. Ovo može biti korisno ako treba da radite sa velikim delovima JSON podataka, ali samo sa određenim podskupom; podaci koji vam nisu potrebni mogu se lako odbaciti.

### Raščlanjivanje u interfejs

Kada unapred znamo kakvu vrstu JSON-a da očekujemo, možemo je analizirati u određenu strukturu. Ali šta ako ne znamo?

Znamo da `interface{}` može biti bilo šta u Go-u, tako da je to najbolji kontejner za čuvanje našeg JSON-a nepoznatog formata. JSON paket koristi `map[string]interface{}` i `[]interface{}` da bi sačuvao sve vrste JSON objekata i nizova. Evo liste JSON relacija mapiranja:

- `bool` predstavlja JSON booleans,
- `float64` predstavlja JSON numbers,
- `string` predstavlja JSON strings,
- `nil` predstavlja JSON null.

Pretpostavimo da imamo sledeće JSON podatke:

```json
b := []byte(`{"Name":"Wednesday","Age":6,"Parents":["Gomez","Morticia"]}`)
```

Sada analiziramo ovaj JSON u `interface{}`:

```go
var f interface{}
err := json.Unmarshal(b, &f)
```

Čuva f mapu, gde su ključevi stringovi, a vrednosti su `interface{}`.

```go
f = map[string]interface{}{
    "Name": "Wednesday",
    "Age":  6,
    "Parents": []interface{}{
        "Gomez",
        "Morticia",
    },
}
```

Dakle, kako pristupamo ovim podacima? Tipska tvrdnja.

```go
m := f.(map[string]interface{})
```

Nakon što je potvrđeno, možete koristiti sledeći kod za pristup podacima:

```go
for k, v := range m {
    switch vv := v.(type) {
    case string:
        fmt.Println(k, "is string", vv)
    case int:
        fmt.Println(k, "is int", vv)
    case float64:
        fmt.Println(k,"is float64",vv)
    case []interface{}:
        fmt.Println(k, "is an array:")
        for i, u := range vv {
            fmt.Println(i, u)
        }
    default:
        fmt.Println(k, "is of a type I don't know how to handle")
    }
}
```

Kao što vidite, sada možemo da analiziramo JSON nepoznatog formata kroz `interface{}` i `type assertion`.

Gore navedeni primer je zvanično rešenje, ali potvrđivanje tipa nije uvek zgodno. Zato preporučujem projekat otvorenog koda pod nazivom `simplejson`, koji je kreirao i održava bitly.

Evo primera kako se ovaj projekat koristi za rad sa JSON-om nepoznatog formata:

```go
js, err := NewJson([]byte(`{
    "test": {
        "array": [1, "2", 3],
        "int": 10,
        "float": 5.150,
        "bignum": 9223372036854775807,
        "string": "simplejson",
        "bool": true
    }
}`))

arr, _ := js.Get("test").Get("array").Array()
i, _ := js.Get("test").Get("int").Int()
ms := js.Get("test").Get("string").MustString()
```

Nije teško videti koliko je ovo zgodno. Pogledajte repozitorijum da biste videli više informacija: <https://github.com/bitly/go-simplejson>.

## Izgradnja JSON-a

U mnogim situacijama, potrebno je da proizvedemo JSON podatke i odgovorimo klijentima. U Go-u, JSON paket ima funkciju koja se zove `Marshal` upravo za to:

```go
func Marshal(v interface{}) ([]byte, error)
```

Pretpostavimo da treba da napravimo listu informacija o serveru. Imamo sledeći primer:

```go
package main

import (
    "encoding/json"
    "fmt"
)

type Server struct {
    ServerName string
    ServerIP   string
}

type Serverslice struct {
    Servers []Server
}

func main() {
    var s Serverslice
    s.Servers = append(s.Servers, Server{ServerName: "Shanghai_VPN", ServerIP: "127.0.0.1"})
    s.Servers = append(s.Servers, Server{ServerName: "Beijing_VPN", ServerIP: "127.0.0.2"})
    
    b, err := json.Marshal(s)
    if err != nil {
        fmt.Println("json err:", err)
    }
    fmt.Println(string(b))
}
```

Izlaz:

```json
{"Servers":[{"ServerName":"Shanghai_VPN","ServerIP":"127.0.0.1"},{"ServerName":"Beijing_VPN","ServerIP":"127.0.0.2"}]}
```

Kao što znate, sva imena polja se pišu velikim slovom, ali ako želite da imena vaših JSON ključeva počinju malim slovom, trebalo bi da koristite `struct tags`. U suprotnom, Go neće proizvesti podatke za interna polja.

```go
type Server struct {
    ServerName string `json:"serverName"`
    ServerIP   string `json:"serverIP"`
}

type Serverslice struct {
    Servers []Server `json:"servers"`
}
```

Nakon ove modifikacije, možemo proizvesti iste JSON podatke kao i ranije.

Evo nekoliko stvari koje treba imati na umu kada pokušavate da kreirate JSON:

- Oznake polja koje sadrže "-" neće biti ispisane.
- Ako oznaka sadrži prilagođeno ime, Go koristi ovo umesto imena polja, kao serverNameu gornjem
  primeru.
- Ako oznaka sadrži omitempty, ovo polje se neće prikazivati ako je vrednost nula.
- Ako je tip polja bool, string, int, int64itd., i njegova oznaka sadrži ",string", Go konvertuje
  ovo polje u odgovarajući JSON tip.

Primer:

```go
type Server struct {
    // ID will not be outputed.
    ID int `json:"-"`

    // ServerName2 will be converted to JSON type.
    ServerName  string `json:"serverName"`
    ServerName2 string `json:"serverName2,string"`

    // If ServerIP is empty, it will not be outputted.
    ServerIP   string `json:"serverIP,omitempty"`
}

s := Server {
    ID:         3,
    ServerName:  `Go "1.0" `,
    ServerName2: `Go "1.0" `,
    ServerIP:   ``,
}
b, _ := json.Marshal(s)
os.Stdout.Write(b)
```

Izlaz:

```json
{"serverName":"Go \"1.0\" ","serverName2":"\"Go \\\"1.0\\\" \""}
```

Funkcija `Marshal` vraća podatke samo kada je uspešno izvršena, pa evo nekoliko stvari koje treba imati na umu:

- JSON podržava samo stringove kao ključeve, tako da ako želite da kodirate mapu, njen tip mora biti map[string]T, gde je T tip u Go-u.
- Tipovi poput kanala, složenih tipova i funkcija ne mogu se kodirati u JSON.
- Ne pokušavajte da kodirate ciklične podatke, to vodi do beskonačne rekurzije.
- Ako je polje pokazivač, Go ispisuje podatke na koje pokazuje, ili u suprotnom ispisuje null ako pokazuje na nil.

U ovom odeljku smo predstavili kako se dekodiraju i kodiraju JSON podaci u Go-u. Takođe smo pogledali jedan projekat treće strane pod nazivom TsimplejsonT koji je koristan za parsiranje JSON-a ali nepoznatog formata. Sve su to korisni koncepti za razvoj veb aplikacija u Go-u.
