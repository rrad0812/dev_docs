
# 10.2 Lokalizovani resursi

[Sadržaj](_00.0-sr.md)

U prethodnom odeljku je opisano kako se podešavaju lokalne vrednosti. Nakon što je lokalna vrednost podešena, potrebno je da se pozabavimo problemom čuvanja informacija koje odgovaraju određenim lokalnim vrednostima. Ove informacije mogu da uključuju: tekstualni sadržaj, vreme i datum, vrednosti valuta, slike, određene datoteke i druge resurse za prikaz. U programu Go, sve ove kontekstualne informacije se čuvaju u JSON formatu na našem serveru, kako bi se pozvale i ubrizgale u naše prikaze kada korisnici iz određenih regiona posete našu veb stranicu. Na primer, sadržaj na engleskom i kineskom jeziku bi se čuvao u datotekama "en.json" i "zh-CN.json", respektivno.

## Lokalizovani tekstualni sadržaj

Običan tekst je najčešći način predstavljanja informacija u veb aplikacijama, i većina vašeg lokalizovanog sadržaja će verovatno biti u ovom obliku. Cilj je da se obezbedi tekstualni sadržaj koji je i idiomatski u odnosu na regionalne izraze i koji se čini prirodnim za strane korisnike vašeg sajta. Jedno rešenje je kreiranje ugnežđene mape lokaliteta, nizova izvornih jezika i njihovih lokalnih pandana. Kada klijenti zatraže stranice sa nekim tekstualnim sadržajem, prvo proveravamo njihov željeni lokalitet, a zatim preuzimamo odgovarajuće nizove sa odgovarajuće mape. Sledeći isečak je jednostavan primer ovog procesa:

```go
package main

import "fmt"

var locales map[string]map[string]string

func main() {
    locales = make(map[string]map[string]string, 2)

    en := make(map[string]string, 10)
    en["pea"] = "pea"
    en["bean"] = "bean"
    locales["en"] = en
    
    cn := make(map[string]string, 10)
    cn["pea"] = "豌豆"
    cn["bean"] = "毛豆"
    locales["zh-CN"] = cn
    
    lang := "zh-CN"

    fmt.Println(msg(lang, "pea"))
    fmt.Println(msg(lang, "bean"))
}

func msg(locale, key string) string {
    if v, ok := locales[locale]; ok {
        if v2, ok := v[key]; ok {
            return v2
        }
    }
    return ""
}
```

Gore navedeni primer postavlja mape prevedenih nizova za različite lokalne vrednosti (u ovom slučaju, kineske i engleske lokalne vrednosti). Mapiramo naše `cn` prevode na iste ključeve engleskog jezika kako bismo mogli da rekonstruišemo našu englesku tekstualnu poruku na kineskom. Ako bismo želeli da promenimo naš tekst na bilo koju drugu lokalnu vrednost koju smo možda implementirali, to bi bilo jednostavno pitanje podešavanja jedne `lang` promenljive.

Jednostavne zamene ključ-vrednost ponekad mogu biti neadekvatni za naše potrebe. Na primer, ako bismo imali frazu kao što je „Imam 30 godina“ gde je 30 promenljiva, kako bismo je lokalizovali U slučajevima poput ovog, možemo kombinovati upotrebu funkcije fmt.Printfda bismo postigli željeni rezultat:

```go
en["how old"] = "I am %d years old"
cn["how old"] = "我今年%d岁了"

fmt.Printf(msg(lang, "how old"), 30)
```

Gore navedeni primer koda je samo u svrhu demonstracije; stvarni podaci o lokalitetu se obično čuvaju u JSON formatu u našoj bazi podataka, što nam omogućava da izvršimo jednostavno json.Unmarshalpopunjavanje lokaliteta mape našim prevodima nizova.

## Lokalizovani datum i vreme

Zbog naših konvencija vremenskih zona, vreme u jednom regionu sveta može se razlikovati od vremena u drugom regionu. Slično tome, način na koji je vreme predstavljeno takođe može varirati od lokala do lokala. Na primer, u kineskom okruženju može biti 2012年10月24日 星期三 23时11分13秒 CST, dok u engleskom može biti: Wed Oct 24 23:11:13 CST 2012. Ne samo da postoje varijacije u jeziku, već postoje i razlike u formatiranju. Dakle, kada je u pitanju lokalizacija datuma i vremena, moramo da se pozabavimo sledećim dvema tačkama:

### Vremenske zone

#### Problemi sa formatiranjem

Direktorijum `$GOROOT/lib/time/package/timeinfo.zip` sadrži lokale koje odgovaraju definicijama vremenskih zona. Da bismo dobili vreme koje odgovara trenutnom lokalu korisnika, prvo treba da koristimo `time.LoadLocation(name string)` da bismo dobili objekat Location koji odgovara našem lokalu, prosleđujući string koji predstavlja lokal, kao što je "Asia/Shanghai" ili "America/Chicago". Zatim možemo da koristimo ovaj objekat `Location` zajedno sa objektom `Time` (dobijenim pozivanjem `time.Now`) da bismo dobili konačno vreme koristeći `In` metodu objekta `Time`. Detaljan pogled na ovaj proces može se videti u nastavku (ovaj primer koristi neke od promenljivih iz gornjeg primera):

```go
en["time_zone"] = "America/Chicago"
cn["time_zone"] = "Asia/Shanghai"

loc, _ := time.LoadLocation(msg(lang, "time_zone"))
t := time.Now()
t = t.In(loc)
fmt.Println(t.Format(time.RFC3339))
```

Formatiranje teksta možemo da obradimo na sličan način da bismo rešili naš problem formatiranja vremena:

```go
en["date_format"]="%Y-%m-%d %H:%M:%S"
cn["date_format"]="%Y年%m月%d日 %H时%M分%S秒"

fmt.Println(date(msg(lang,"date_format"),t))
```

## Lokalizovana vrednost valute

Očigledno je da se valuta razlikuje od regiona do regiona. Možemo je tretirati na isti način kao što smo tretirali naše datume:

```go
en["money"] ="USD %d"
cn["money"] ="￥%d元"

fmt.Println(money_format(msg(lang,"money"),100))

func money_format(fomat string, money int64) string{
    return fmt.Sprintf(fomat, money)
}
```

## Lokalizacija prikaza i resursa

Možemo da prikažemo prilagođene prikaze sa različitim slikama, css-om, js-om i drugim statičkim resursima u zavisnosti od trenutne lokalizacije. Jedan od načina da se to postigne jeste organizovanje ovih datoteka u njihove odgovarajuće lokalizacije. Evo primera:

```sh
views
|--en  //English Templates
    |--images     //store picture information
    |--js         //JS files 
    |--css        //CSS files
    index.tpl     //User Home
    login.tpl     //Log Home
|--zh-CN //Chinese Templates
    |--images
    |--js
    |--css
    index.tpl
    login.tpl
```

Sa ovom strukturom direktorijuma, možemo da prikažemo prikaze specifične za lokalizaciju ovako:

```go
s1, _ := template.ParseFiles("views" + lang + "index.tpl")
VV.Lang = lang
s1.Execute(os.Stdout, VV)
```

Resursi navedeni u index.tpl datoteci mogu se tretirati na sledeći način:

```js
// js file
<script type="text/javascript" src="views/{{.VV.Lang}}/js/jquery/jquery-1.8.0.min.js"></script>
```

```css
// css file
<link href="views/{{.VV.Lang}}/css/bootstrap-responsive.min.css" rel="stylesheet">
// Picture files
<img src="views/{{.VV.Lang}}/images/btn.png">
```

Sa dinamičkim prikazima i načinom na koji smo lokalizovali naše resurse, moći ćemo da dodamo više lokalizacija bez mnogo napora.

## Rezime

Ovaj odeljak je opisao kako se koriste i čuvaju lokalni resursi. Naučili smo da za ovo možemo koristiti funkcije konverzije i interpolaciju nizova i videli smo da mape mogu biti efikasan način čuvanja podataka specifičnih za lokalne standarde. Za ovo drugo, mogli bismo jednostavno izdvojiti odgovarajuće informacije o lokalnim standardima kada je to potrebno - ako je to bio tekstualni sadržaj koji smo želeli, naši mapirani prevodi i idiomi mogli bismo direktno proslediti na izlaz. Ako je u pitanju bilo nešto sofisticiranije poput vremena ili valute, jednostavno smo koristili funkciju `fmt.Printf` da ga prethodno formatiramo. Lokalizacija naših prikaza i resursa bila je najlakši slučaj i jednostavno je uključivala organizovanje naših datoteka u njihove odgovarajuće lokalne standarde, a zatim njihovo referenciranje iz njihovih relativnih putanja lokalnih standarda.
