## 7.1 XML

XML je često korišćeni format za komunikaciju podataka u veb servisima. Danas, on preuzima sve važniju ulogu u veb razvoju. U ovom odeljku, predstavićemo vam kako da radite sa XML-om putem Go standardne biblioteke.

Neću pokušavati da vas naučim sintaksi ili konvencijama XML-a. Za to, molim vas pročitajte više dokumentacije o samom XML-u. Fokusiraćemo se samo na to kako kodirati i dekodirati XML datoteke u Go-u.

Pretpostavimo da radite u IT sektoru i da morate da se bavite sledećom XML konfiguracionom datotekom:

```xml
<?xml version="1.0" encoding="utf-8"?>
<servers version="1">
    <server>
        <serverName>Shanghai_VPN</serverName>
        <serverIP>127.0.0.1</serverIP>
    </server>
    <server>
        <serverName>Beijing_VPN</serverName>
        <serverIP>127.0.0.2</serverIP>
    </server>
</servers>
```

Gore navedeni XML dokument sadrži dve vrste informacija o vašem serveru: "serverName" i "serverIP". Koristićemo ovaj dokument u sledećim primerima.

### Raščlanjivanje XML-a

Kako da analiziramo ovaj XML dokument? Možemo koristiti `Unmarshal` funkciju u Go `xml` paketu da bismo to uradili.

```go
func Unmarshal(data []byte, v interface{}) error
```

Parametar `data` prima tok podataka iz XML izvora i v predstavlja strukturu u koju želite da ispišete analizirani XML. To je interfejs, što znači da možete konvertovati XML u bilo koju strukturu koju želite. Ovde ćemo govoriti samo o tome kako konvertovati iz XML-a u `struct` tip, jer dele slične strukture stabla.

Primer koda:

```go
package main

import (
    "encoding/xml"
    "fmt"
    "io/ioutil"
    "os"
)

type Recurlyservers struct {
    XMLName     xml.Name `xml:"servers"`
    Version     string   `xml:"version,attr"`
    Svs         []server `xml:"server"`
    Description string   `xml:",innerxml"`
}

type server struct {
    XMLName    xml.Name `xml:"server"`
    ServerName string   `xml:"serverName"`
    ServerIP   string   `xml:"serverIP"`
}

func main() {
    file, err := os.Open("servers.xml") // For read access.     
    if err != nil {
        fmt.Printf("error: %v", err)
        return
    }
    defer file.Close()
    data, err := ioutil.ReadAll(file)
    if err != nil {
        fmt.Printf("error: %v", err)
        return
    }
    v := Recurlyservers{}
    err = xml.Unmarshal(data, &v)
    if err != nil {
        fmt.Printf("error: %v", err)
        return
    }

    fmt.Println(v)
}
```

XML je zapravo struktura podataka tipa stabla i možemo definisati veoma sličnu strukturu koristeći strukture u programu Go, a zatim je koristiti sa `xml.Unmarshal` za konvertovanje iz XML-a u naš objekat strukture. Primer koda će ispisati sledeći sadržaj:

```xml
{{ servers} 1 [{{ server} Shanghai_VPN 127.0.0.1} {{ server} Beijing_VPN 127.0.0.2}]
<server>
    <serverName>Shanghai_VPN</serverName>
    <serverIP>127.0.0.1</serverIP>
</server>
<server>
    <serverName>Beijing_VPN</serverName>
    <serverIP>127.0.0.2</serverIP>
</server>
}
```

Koristimo `xml.Unmarshal` za parsiranje XML dokumenta na odgovarajući objekat strukture. Trebalo bi da vidite da imamo nešto slično xml:"serverName"u našoj strukturi. Ovo je karakteristika struktura koja se poziva struct tags da pomogne pri refleksiji. Pogledajmo ponovo `Unmarshal` definiciju:

```go
func Unmarshal(data []byte, v interface{}) error
```

Prvi argument je XML tok podataka. Drugi argument je tip skladištenja i podržava tipove struktura, isečaka i stringova. Go-ov XML paket koristi refleksiju za mapiranje podataka, tako da sva polja u "v" treba da se izvezu. Međutim, ovo stvara problem: kako zna koje XML polje odgovara mapiranom polju strukture? Odgovor je da XML parser analizira podatke određenim redosledom. Biblioteka će prvo pokušati da pronađe odgovarajuću oznaku strukture. Ako se podudaranje ne može pronaći, onda pretražuje imena polja strukture. Imajte na umu da su sve oznake, imena polja i XML elementi osetljivi na velika i mala slova, tako da morate da se uverite da postoji jednaka podudarnost da bi mapiranje uspelo.

Go-ov mehanizam refleksije vam omogućava da koristite ove informacije o oznakama za refleksiju XML podataka na objekat strukture. Ako želite da saznate više o refleksiji u Go-u, pročitajte dokumentaciju paketa o oznakama strukture i refleksiji.

Evo nekih pravila pri korišćenju `xml` paketa za parsiranje XML dokumenata u strukture:

- Ako je tip polja `string` ili `[]byte` sa oznakom ",innerxml", Unmarshal će mu dodeliti sirove XML podatke, kao u "Description" u gornjem primeru:

Šangaj_VPN 127.0.0.1 Peking_VPN 127.0.0.2

- Ako se pozove polje `XMLName` tipa `xml.Name`, dobija se ime elementa, kao "servers" u gornjem primeru.

- Ako oznaka polja sadrži odgovarajuće ime elementa, onda dobija i ime elementa, kao "servername" u "servers" u gornjem primeru.

- Ako oznaka polja sadrži ",attr", onda dobija atribut odgovarajućeg elementa, kao version u gornjem primeru.

- Ako oznaka polja sadrži nešto poput "a>b>c", dobija vrednost elementa c čvora b čvora a.

- Ako oznaka polja sadrži "=", onda ono ne dobija ništa.

- Ako oznaka polja sadrži "any", onda dobija sve podređene elemente koji ne odgovaraju ostalim pravilima.

- Ako XML elementi imaju jedan ili više komentara, svi ovi komentari će biti dodati prvom polju koje ima oznaku a koje sadrži "comments". Ovaj tip polja može biti `string` ili `[]byte`. Ako ova vrsta polja ne postoji, svi komentari se odbacuju.

Ova pravila vam govore kako da definišete oznake u strukturama. Kada razumete ova pravila, mapiranje XML-a na strukture biće jednostavno kao i primer koda iznad. Pošto oznake i XML elementi imaju jedan-na-jedan korespondenciju, možemo koristiti i isečke da predstavite više elemenata na istom nivou.

Imajte na umu da sva polja u strukturama treba da budu izvezena (velikim slovima) kako bi se podaci pravilno analizirali.

### Izgradnja XML-a

Šta ako želimo da napravimo XML dokument umesto da ga parsiramo? Kako to da uradimo u Go-u? Nije iznenađujuće da `xml` paket pruža dve funkcije, `Marshal` i `MarshalIndent`, gde druga funkcija automatski uvlači maršalirani XML dokument. Njihova definicija je sledeća:

```go
func Marshal(v interface{}) ([]byte, error)
func MarshalIndent(v interface{}, prefix, indent string) ([]byte, error)
```

Prvi argument u obe ove funkcije je za čuvanje maršaliranog XML toka podataka.

Pogledajmo primer da vidimo kako ovo funkcioniše:

```go
package main

import (
    "encoding/xml"
    "fmt"
    "os"
)

type Servers struct {
    XMLName xml.Name `xml:"servers"`
    Version string   `xml:"version,attr"`
    Svs     []server `xml:"server"`
}

type server struct {
    ServerName string `xml:"serverName"`
    ServerIP   string `xml:"serverIP"`
}

func main() {
    v := &Servers{Version: "1"}
    v.Svs = append(v.Svs, server{"Shanghai_VPN", "127.0.0.1"})
    v.Svs = append(v.Svs, server{"Beijing_VPN", "127.0.0.2"})
    output, err := xml.MarshalIndent(v, "  ", "    ")
    if err != nil {
        fmt.Printf("error: %v\n", err)
    }
    os.Stdout.Write([]byte(xml.Header))

    os.Stdout.Write(output)
}
```

Gore navedeni primer ispisuje sledeće informacije:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<servers version="1">
<server>
    <serverName>Shanghai_VPN</serverName>
    <serverIP>127.0.0.1</serverIP>
</server>
<server>
    <serverName>Beijing_VPN</serverName>
    <serverIP>127.0.0.2</serverIP>
</server>
</servers>
```

Kao što smo prethodno definisali, razlog zašto jesmo `os.Stdout.Write([]byte(xml.Header))` je taj što ni `xml.MarshalIndent`, ni `xml.Marshal` ne prikazuju XML zaglavlja sama po sebi, pa ih moramo eksplicitno ispisati da bismo ispravno proizveli XML dokumente.

Ovde možemo videti da `Marshal` takođe prima parametre tipa `interface{}`. Dakle, koja su pravila prilikom maršaliranja u XML dokument?

- Ako je v string ili isečak, ispisuje sve elemente kao vrednost.
- Ako je v pokazivač, ispisuje sadržaj na koji v pokazuje, ne ispisuje ništa kada je v jednako `nil`.
- Ako je v interfejs, on se bavi i tim interfejsom.
- Ako je v jedan od drugih tipova, ispisuje vrednost tog tipa.

Pa kako `xml.Marshal` određuje ime elemenata? Prati sledeća pravila:

- Ako je v struktura, ona definiše ime u oznaci XMLName.
- Ime polja je XMLName, a tip je xml.Name.
- Oznaka polja u strukturi.
- Ime polja u strukturi.
- Ukucajte ime maršala.

Zatim moramo da shvatimo kako da postavimo oznake da bismo napravili konačni XML dokument.

- XMLName neće biti odštampan.
- Polja koja sadrže oznake "-" neće biti odštampana.
- Ako oznaka sadrži "name,attr", koristi ime kao ime atributa i vrednost polja kao vrednost, kao version u gornjem primeru.
- Ako oznaka sadrži ",attr", koristi naziv polja kao naziv atributa, a vrednost polja kao njegovu vrednost.
- Ako oznaka sadrži ",chardata", ona ispisuje karakterne podatke umesto elementa.
- Ako oznaka sadrži ",innerxml", ona ispisuje sirovu vrednost.
- Ako oznaka sadrži ",comment", ispisuje se kao komentar bez izlaza, tako da ne možete imati "–" u njenoj vrednosti.
- Ako oznaka sadrži "omitempty", ovo polje se izostavlja ako je njegova vrednost nula-vrednost, uključujući false, 0, nil pokazivač ili nil interfejs, nultu dužinu niza, iseečka, mape i stringa.
- Ako oznaka sadrži "a>b>c", ispisuje tri elementa gde a sadrži b, a b sadrži c, kao u sledećem kodu:

FirstName string   `xml:"name>first"`
LastName  string   `xml:"name>last"`

```xml
<name>
<first>Asta</first>
<last>Xie</last>
</name>
```

Možda ste primetili da su strukturne oznake veoma korisne za rad sa XML-om, a isto važi i za ostale formate podataka o kojima ćemo govoriti u narednim odeljcima. Ako i dalje imate problema sa radom sa strukturnim oznakama, verovatno bi trebalo da pročitate više dokumentacije o njima pre nego što pređete na sledeći odeljak.

## 7.2 JSON

JSON (JavaScript Object Notation) je lagan jezik za razmenu podataka koji se zasniva na tekstualnom opisu. Njegove prednosti uključuju to što je samoopisan, lako se razume itd. Iako je podskup JavaScript-a, JSON koristi drugačiji tekstualni format, što ga čini nezavisnim jezikom. JSON je sličan jezicima C porodice.

Najveća razlika između JSON-a i XML-a je u tome što je XML kompletan jezik za označavanje, dok JSON nije. JSON je manji i brži od XML-a, stoga ga je mnogo lakše i brže analizirati u pregledačima, što je jedan od razloga zašto mnoge otvorene platforme biraju da koriste JSON kao svoj jezik interfejsa za razmenu podataka.

Pošto JSON postaje sve važniji u veb razvoju, hajde da pogledamo nivo podrške koju Go ima za JSON. Videćete da Go-ova standardna biblioteka ima veoma dobru podršku za kodiranje i dekodiranje JSON-a.

Ovde koristimo JSON da predstavimo primer iz prethodnog odeljka:

```json
{"servers":[{"serverName":"Shanghai_VPN","serverIP":"127.0.0.1"},{"serverName":"Beijing_VPN","serverIP":"127.0.0.2"}]}
```

Ostatak ovog odeljka će koristiti ove JSON podatke za uvođenje JSON koncepata u Go.

### Raščlanjivanje JSON-a

#### Raščlanjivanje u strukturu

Pretpostavimo da imamo JSON u gornjem primeru. Kako možemo da analiziramo ove podatke i mapiramo ih u strukturu u Go-u? Go pruža sledeću funkciju upravo za tu svrhu:

```go
func Unmarshal(data []byte, v interface{}) error
```

Ovu funkciju možemo koristiti ovako:

```go
package main

import (
    "encoding/json"
    "fmt"
)

type Server struct {
    ServerName string
    ServerIP   string
}

type Serverslice struct {
    Servers []Server
}

func main() {
    var s Serverslice
    str := `{"servers":[{"serverName":"Shanghai_VPN","serverIP":"127.0.0.1"},{"serverName":"Beijing_VPN","serverIP":"127.0.0.2"}]}`
    json.Unmarshal([]byte(str), &s)
    fmt.Println(s)
}
```

U gornjem primeru, definisali smo odgovarajuće strukture u Go-u za naš JSON, koristeći slice za niz JSON objekata i naziv polja kao naše JSON ključeve. Ali kako Go zna koji JSON objekat odgovara kojoj određenoj strukturi? Pretpostavimo da imamo ključ koji se zove Foo u JSON-u. Kako da pronađemo njegovo odgovarajuće polje?

- Prvo, Go pokušava da pronađe (velikim slovima) izvezeno polje čija oznaka sadrži Foo.
- Ako se ne može pronaći podudaranje, potražite polje čije je ime Foo.
- Ako i dalje nema podudaranja, potražite nešto poput FOO ili FoO, zanemarujući razliku između velikih i malih slova.

Možda ste primetili da sva polja koja će biti dodeljena treba da budu izvezena, a Go dodeljuje samo polja koja se mogu pronaći, ignorišući sva ostala. Ovo može biti korisno ako treba da radite sa velikim delovima JSON podataka, ali samo sa određenim podskupom; podaci koji vam nisu potrebni mogu se lako odbaciti.

#### Raščlanjivanje u interfejs

Kada unapred znamo kakvu vrstu JSON-a da očekujemo, možemo je analizirati u određenu strukturu. Ali šta ako ne znamo?

Znamo da `interface{}` može biti bilo šta u Go-u, tako da je to najbolji kontejner za čuvanje našeg JSON-a nepoznatog formata. JSON paket koristi `map[string]interface{}` i `[]interface{}` da bi sačuvao sve vrste JSON objekata i nizova. Evo liste JSON relacija mapiranja:

- `bool` predstavlja JSON booleans,
- `float64` predstavlja JSON numbers,
- `string` predstavlja JSON strings,
- `nil` predstavlja JSON null.

Pretpostavimo da imamo sledeće JSON podatke:

```json
b := []byte(`{"Name":"Wednesday","Age":6,"Parents":["Gomez","Morticia"]}`)
```

Sada analiziramo ovaj JSON u `interface{}`:

```go
var f interface{}
err := json.Unmarshal(b, &f)
```

Čuva f mapu, gde su ključevi stringovi, a vrednosti su `interface{}`.

```go
f = map[string]interface{}{
    "Name": "Wednesday",
    "Age":  6,
    "Parents": []interface{}{
        "Gomez",
        "Morticia",
    },
}
```

Dakle, kako pristupamo ovim podacima? Tipska tvrdnja.

```go
m := f.(map[string]interface{})
```

Nakon što je potvrđeno, možete koristiti sledeći kod za pristup podacima:

```go
for k, v := range m {
    switch vv := v.(type) {
    case string:
        fmt.Println(k, "is string", vv)
    case int:
        fmt.Println(k, "is int", vv)
    case float64:
        fmt.Println(k,"is float64",vv)
    case []interface{}:
        fmt.Println(k, "is an array:")
        for i, u := range vv {
            fmt.Println(i, u)
        }
    default:
        fmt.Println(k, "is of a type I don't know how to handle")
    }
}
```

Kao što vidite, sada možemo da analiziramo JSON nepoznatog formata kroz `interface{}` i `type assertion`.

Gore navedeni primer je zvanično rešenje, ali potvrđivanje tipa nije uvek zgodno. Zato preporučujem projekat otvorenog koda pod nazivom `simplejson`, koji je kreirao i održava bitly.

Evo primera kako se ovaj projekat koristi za rad sa JSON-om nepoznatog formata:

```go
js, err := NewJson([]byte(`{
    "test": {
        "array": [1, "2", 3],
        "int": 10,
        "float": 5.150,
        "bignum": 9223372036854775807,
        "string": "simplejson",
        "bool": true
    }
}`))

arr, _ := js.Get("test").Get("array").Array()
i, _ := js.Get("test").Get("int").Int()
ms := js.Get("test").Get("string").MustString()
```

Nije teško videti koliko je ovo zgodno. Pogledajte repozitorijum da biste videli više informacija: <https://github.com/bitly/go-simplejson>.

### Izgradnja JSON-a

U mnogim situacijama, potrebno je da proizvedemo JSON podatke i odgovorimo klijentima. U Go-u, JSON paket ima funkciju koja se zove `Marshal` upravo za to:

```go
func Marshal(v interface{}) ([]byte, error)
```

Pretpostavimo da treba da napravimo listu informacija o serveru. Imamo sledeći primer:

```go
package main

import (
    "encoding/json"
    "fmt"
)

type Server struct {
    ServerName string
    ServerIP   string
}

type Serverslice struct {
    Servers []Server
}

func main() {
    var s Serverslice
    s.Servers = append(s.Servers, Server{ServerName: "Shanghai_VPN", ServerIP: "127.0.0.1"})
    s.Servers = append(s.Servers, Server{ServerName: "Beijing_VPN", ServerIP: "127.0.0.2"})
    
    b, err := json.Marshal(s)
    if err != nil {
        fmt.Println("json err:", err)
    }
    fmt.Println(string(b))
}
```

Izlaz:

```json
{"Servers":[{"ServerName":"Shanghai_VPN","ServerIP":"127.0.0.1"},{"ServerName":"Beijing_VPN","ServerIP":"127.0.0.2"}]}
```

Kao što znate, sva imena polja se pišu velikim slovom, ali ako želite da imena vaših JSON ključeva počinju malim slovom, trebalo bi da koristite `struct tags`. U suprotnom, Go neće proizvesti podatke za interna polja.

```go
type Server struct {
    ServerName string `json:"serverName"`
    ServerIP   string `json:"serverIP"`
}

type Serverslice struct {
    Servers []Server `json:"servers"`
}
```

Nakon ove modifikacije, možemo proizvesti iste JSON podatke kao i ranije.

Evo nekoliko stvari koje treba imati na umu kada pokušavate da kreirate JSON:

- Oznake polja koje sadrže "-" neće biti ispisane.
- Ako oznaka sadrži prilagođeno ime, Go koristi ovo umesto imena polja, kao serverNameu gornjem
  primeru.
- Ako oznaka sadrži omitempty, ovo polje se neće prikazivati ako je vrednost nula.
- Ako je tip polja bool, string, int, int64itd., i njegova oznaka sadrži ",string", Go konvertuje
  ovo polje u odgovarajući JSON tip.

Primer:

```go
type Server struct {
    // ID will not be outputed.
    ID int `json:"-"`

    // ServerName2 will be converted to JSON type.
    ServerName  string `json:"serverName"`
    ServerName2 string `json:"serverName2,string"`

    // If ServerIP is empty, it will not be outputted.
    ServerIP   string `json:"serverIP,omitempty"`
}

s := Server {
    ID:         3,
    ServerName:  `Go "1.0" `,
    ServerName2: `Go "1.0" `,
    ServerIP:   ``,
}
b, _ := json.Marshal(s)
os.Stdout.Write(b)
```

Izlaz:

```json
{"serverName":"Go \"1.0\" ","serverName2":"\"Go \\\"1.0\\\" \""}
```

Funkcija `Marshal` vraća podatke samo kada je uspešno izvršena, pa evo nekoliko stvari koje treba imati na umu:

- JSON podržava samo stringove kao ključeve, tako da ako želite da kodirate mapu, njen tip mora biti map[string]T, gde je T tip u Go-u.
- Tipovi poput kanala, složenih tipova i funkcija ne mogu se kodirati u JSON.
- Ne pokušavajte da kodirate ciklične podatke, to vodi do beskonačne rekurzije.
- Ako je polje pokazivač, Go ispisuje podatke na koje pokazuje, ili u suprotnom ispisuje null ako pokazuje na nil.

U ovom odeljku smo predstavili kako se dekodiraju i kodiraju JSON podaci u Go-u. Takođe smo pogledali jedan projekat treće strane pod nazivom TsimplejsonT koji je koristan za parsiranje JSON-a ali nepoznatog formata. Sve su to korisni koncepti za razvoj veb aplikacija u Go-u.

## 7.3 Regexp

Regularni izrazi (`Regexp`) su komplikovan, ali moćan alat za uparivanje obrazaca i manipulaciju tekstom. Iako ne funkcioniše tako dobro kao čisto uparivanje teksta, fleksibilniji je. Na osnovu svoje sintakse, možete filtrirati skoro svaku vrstu teksta iz izvornog sadržaja. Ako treba da prikupite podatke u veb razvoju, nije teško koristiti `Regexp` za preuzimanje značajnih podataka.

Go ima `regexp` paket koji pruža zvaničnu podršku za `regexp`. Ako ste već koristili `regexp` u drugim programskim jezicima, trebalo bi da ste upoznati sa njim. Imajte na umu da je Go implementirao RE2 standard, osim za \C. Za više detalja, pratite ovaj link: <http://code.google.com/p/re2/wiki/Syntax>.

Go-ov `strings` paket zapravo može da obavlja mnoge poslove kao što su pretraživanje (`Contains`, `Index`), zamena (`Replace`), parsiranje (`Split`, `Join`) itd., i brži je od Regexp-a. Međutim, sve su to trivijalne operacije. Ako želite da pretražujete string koji nije osetljiv na velika i mala slova, Regexp bi trebalo da bude vaš najbolji izbor. Dakle, ako je `strings` paket dovoljan za vaše potrebe, jednostavno ga koristite jer je jednostavan za korišćenje i čitanje; ako treba da izvršite naprednije operacije, koristite `Regexp`.

Ako se sećate validacije formulara iz prethodnih odeljaka, koristili smo `Regexp` da bismo proverili validnost korisnički unetih informacija. Imajte na umu da su svi znakovi UTF-8. Hajde da saznamo više o Go `regexp` paketu!

### Match

Paket `regexp` ima 3 funkcije za podudaranje: ako se podudara sa šablonom, onda vraća vrednost `true`, u suprotnom vraća vrednost `false`.

```go
func Match(pattern string, b []byte) (matched bool, error error)
func MatchReader(pattern string, r io.RuneReader) (matched bool, error error)
func MatchString(pattern string, s string) (matched bool, error error)
```

Sve 3 funkcije proveravaju da li se `pattern` podudara sa ulaznim izvorom, vraćajući vrednost `true` ako se podudara. Međutim, ako vaš regularni izraz ima sintaksičke greške, vratiće grešku. 3 ulazna izvora ovih funkcija su `slice of byte`, `RuneReader` i `string`.

Evo primera kako da potvrdite IP adresu:

```go
func IsIP(ip string) (b bool) {
    if m, _ := regexp.MatchString("^[0-9]{1,3}\\.[0-9]{1,3}\\.[0-9]{1,3}\\.[0-9]{1,3}$", ip); !m {
        return false
    }
    return true
}
```

Kao što vidite, korišćenje šablona u `regexp` paketu nije toliko drugačije. Evo još jednog primera o proveri da li je korisnički unos validan:

```go
func main() {
    if len(os.Args) == 1 {
        fmt.Println("Usage: regexp [string]")
        os.Exit(1)
    } else if m, _ := regexp.MatchString("^[0-9]+$", os.Args[1]); m {
        fmt.Println("Number")
    } else {
        fmt.Println("Not number")
    }
}
```

U gornjim primerima koristimo `Match(Reader|String` da proverimo da li je sadržaj validan, ali su svi jednostavni za korišćenje.

### Filter

Režim podudaranja može da verifikuje sadržaj, ali ne može da ga iseče, filtrira ili prikuplja podatke iz njega. Ako želite to da uradite, morate da koristite složeni režim `Regexp`-a.

Recimo da treba da napišemo pretraživač. Evo primera kada morate da koristite `Regexp` za filtriranje i isecanje podataka.

```go
package main

import (
    "fmt"
    "io/ioutil"
    "net/http"
    "regexp"
    "strings"
)

func main() {
    resp, err := http.Get("http://www.baidu.com")
    if err != nil {
        fmt.Println("http get error.")
    }
    defer resp.Body.Close()
    
    body, err := ioutil.ReadAll(resp.Body)
    if err != nil {
        fmt.Println("http read error")
        return
    }

    src := string(body)

    // Convert HTML tags to lower case.
    re, _ := regexp.Compile("\\<[\\S\\s]+?\\>")
    src = re.ReplaceAllStringFunc(src, strings.ToLower)

    // Remove STYLE.
    re, _ = regexp.Compile("\\<style[\\S\\s]+?\\</style\\>")
    src = re.ReplaceAllString(src, "")

    // Remove SCRIPT.
    re, _ = regexp.Compile("\\<script[\\S\\s]+?\\</script\\>")
    src = re.ReplaceAllString(src, "")

    // Remove all HTML code in angle brackets, and replace with newline.
    re, _ = regexp.Compile("\\<[\\S\\s]+?\\>")
    src = re.ReplaceAllString(src, "\n")

    // Remove continuous newline.
    re, _ = regexp.Compile("\\s{2,}")
    src = re.ReplaceAllString(src, "\n")

    fmt.Println(strings.TrimSpace(src))
}
```

U ovom primeru, koristimo kompajliranje kao prvi korak za složeni režim. Kod proverava da li je vaša regularna sintaksa ispravna, a zatim vraća vrednost `Regexp` za parsiranje sadržaja u drugim operacijama.

Evo nekih funkcija za analizu vaše Regexp sintakse:

```go
func Compile(expr string) (*Regexp, error)
func CompilePOSIX(expr string) (*Regexp, error)
func MustCompile(str string) *Regexp
func MustCompilePOSIX(str string) *Regexp
```

Razlika između `ComplePOSIX` i `Compile` je u tome što prvi mora da koristi POSIX sintaksu koja je najduža pretraga krajnje levo, a drugi je samo pretraga krajnje levo. Na primer, za `Regexp` `[a-z]`{2,4}` i content "aa09aaa88aaaa", `CompilePOSIX` vraća se aaaaa dok `Compile` vraća prefiks `aa`.`Must` znači paniku kada sintaksa `Regexp` nije ispravna, u suprotnom vraća grešku.

Sada kada znamo kako da kreiramo novi Regexp, hajde da vidimo kako nam metode koje pruža ova struktura mogu pomoći u radu sa sadržajem:

```go
func (re *Regexp) Find(b []byte) []byte
func (re *Regexp) FindAll(b []byte, n int) [][]byte
func (re *Regexp) FindAllIndex(b []byte, n int) [][]int
func (re *Regexp) FindAllString(s string, n int) []string
func (re *Regexp) FindAllStringIndex(s string, n int) [][]int
func (re *Regexp) FindAllStringSubmatch(s string, n int) [][]string
func (re *Regexp) FindAllStringSubmatchIndex(s string, n int) [][]int
func (re *Regexp) FindAllSubmatch(b []byte, n int) [][][]byte
func (re *Regexp) FindAllSubmatchIndex(b []byte, n int) [][]int
func (re *Regexp) FindIndex(b []byte) (loc []int)
func (re *Regexp) FindReaderIndex(r io.RuneReader) (loc []int)
func (re *Regexp) FindReaderSubmatchIndex(r io.RuneReader) []int
func (re *Regexp) FindString(s string) string
func (re *Regexp) FindStringIndex(s string) (loc []int)
func (re *Regexp) FindStringSubmatch(s string) []string
func (re *Regexp) FindStringSubmatchIndex(s string) []int
func (re *Regexp) FindSubmatch(b []byte) [][]byte
func (re *Regexp) FindSubmatchIndex(b []byte) []int
```

Ovih 18 metoda uključuje identične funkcije za različite ulazne izvore (bajtni isečak, string i io.RuneReader), tako da možemo zaista pojednostaviti ovu listu ignorišući ulazne izvore na sledeći način:

```go
func (re *Regexp) Find(b []byte) []byte
func (re *Regexp) FindAll(b []byte, n int) [][]byte
func (re *Regexp) FindAllIndex(b []byte, n int) [][]int
func (re *Regexp) FindAllSubmatch(b []byte, n int) [][][]byte
func (re *Regexp) FindAllSubmatchIndex(b []byte, n int) [][]int
func (re *Regexp) FindIndex(b []byte) (loc []int)
func (re *Regexp) FindSubmatch(b []byte) [][]byte
func (re *Regexp) FindSubmatchIndex(b []byte) []int
```

Primer koda:

```go
package main

import (
    "fmt"
    "regexp"
)

func main() {
    a := "I am learning Go language"

    re, _ := regexp.Compile("[a-z]{2,4}")

    // Find the first match.
    one := re.Find([]byte(a))
    fmt.Println("Find:", string(one))

    // Find all matches and save to a slice, n less than 0 means return all matches, indicates length of slice if it's greater than 0.
    all := re.FindAll([]byte(a), -1)
    fmt.Println("FindAll", all)

    // Find index of first match, start and end position.
    index := re.FindIndex([]byte(a))
    fmt.Println("FindIndex", index)

    // Find index of all matches, the n does same job as above.
    allindex := re.FindAllIndex([]byte(a), -1)
    fmt.Println("FindAllIndex", allindex)

    re2, _ := regexp.Compile("am(.*)lang(.*)")

    // Find first submatch and return array, the first element contains all elements, the second element contains the result of first (), the third element contains the result of second ().
    // Output: 
    // the first element: "am learning Go language"
    // the second element: " learning Go ", notice spaces will be outputed as well.
    // the third element: "uage"
    submatch := re2.FindSubmatch([]byte(a))
    fmt.Println("FindSubmatch", submatch)
    for _, v := range submatch {
        fmt.Println(string(v))
    }

    // Same as FindIndex().
    submatchindex := re2.FindSubmatchIndex([]byte(a))
    fmt.Println(submatchindex)

    // FindAllSubmatch, find all submatches.
    submatchall := re2.FindAllSubmatch([]byte(a), -1)
    fmt.Println(submatchall)

    // FindAllSubmatchIndex,find index of all submatches.
    submatchallindex := re2.FindAllSubmatchIndex([]byte(a), -1)
    fmt.Println(submatchallindex)
}
```

Kao što smo već pomenuli, Regexp takođe ima 3 metode za uparivanje. One rade potpuno istu stvar kao i izvezene funkcije. U stvari, te izvezene funkcije zapravo pozivaju ove metode "ispod haube":

```go
func (re *Regexp) Match(b []byte) bool
func (re *Regexp) MatchReader(r io.RuneReader) bool
func (re *Regexp) MatchString(s string) bool

Zatim, da vidimo kako da zamenimo stringove koristeći Regexp:

func (re *Regexp) ReplaceAll(src, repl []byte) []byte
func (re *Regexp) ReplaceAllFunc(src []byte, repl func([]byte) []byte) []byte
func (re *Regexp) ReplaceAllLiteral(src, repl []byte) []byte
func (re *Regexp) ReplaceAllLiteralString(src, repl string) string
func (re *Regexp) ReplaceAllString(src, repl string) string
func (re *Regexp) ReplaceAllStringFunc(src string, repl func(string) string) string
```

Ovo se koristi u primeru sa pretraživanjem, tako da ovde nećemo dalje objašnjavati.

Hajde da pogledamo definiciju `Expand`:

```go
func (re *Regexp) Expand(dst []byte, template []byte, src []byte, match []int) []byte
func (re *Regexp) ExpandString(dst []byte, template string, src string, match []int) []byte
```

Pa kako da koristimo `Expand`?

```go
func main() {
    src := []byte(`
        call hello alice
        hello bob
        call hello eve
    `)
    pat := regexp.MustCompile(`(?m)(call)\s+(?P<cmd>\w+)\s+(?P<arg>.+)\s*$`)
    res := []byte{}
    for _, s := range pat.FindAllSubmatchIndex(src, -1) {
        res = pat.Expand(res, []byte("$cmd('$arg')\n"), src, s)
    }
    fmt.Println(string(res))
}
```

U ovom trenutku, naučili ste ceo `regexp` paket Go jezika. Nadam se da možete razumeti više proučavajući primere ključnih metoda, kako biste mogli sami da uradite nešto zanimljivo.

## 7.4 Šabloni

### Šta je šablon?

Nadam se da ste upoznati sa MVC (Model, View, Controller) modelom dizajna, gde modeli obrađuju podatke, views prikazuju rezultate i konačno, kontroleri obrađuju zahteve korisnika. Za views, mnogi dinamički jezici generišu podatke pisanjem koda u statičkim HTML datotekama. Na primer, JSP se implementira umetanjem `<%=....=%>`, PHP umetanjem `<?php.....?>`, itd.

Sledeće prikazuje mehanizam šablona:

![7.1](images/7.4.template.png)
Slika 7.1 Mehanizam šablona

Većina sadržaja kojim veb aplikacije odgovaraju klijentima je statički, a dinamički delovi su obično veoma mali. Na primer, ako treba da prikažete listu korisnika koji su posetili stranicu, samo korisničko ime bi bilo dinamičko. Stil liste ostaje isti. Kao što vidite, šabloni su korisni za ponovnu upotrebu statičkog sadržaja.

### Šabloniranje u Gou

U programu Go imamo `template` paket koji pomaže u radu sa šablonima. Možemo koristiti funkcije poput `Parse`, `ParseFile` i `Execute` da učitamo šablone iz običnog teksta ili datoteka, a zatim procenimo dinamičke delove, kao što je prikazano na slici 7.1.

Primer:

```go
func handler(w http.ResponseWriter, r *http.Request) {
    t := template.New("some template") // Create a template.
    t, _ = t.ParseFiles("tmpl/welcome.html", nil)  // Parse template file.
    user := GetUser() // Get current user infomration.
    t.Execute(w, user)  // merge.
}
```

Kao što vidite, veoma je lako koristiti, učitavati i prikazivati podatke u šablonima u Go-u, baš kao i u drugim programskim jezicima.

Radi praktičnosti, u našim primerima ćemo koristiti sledeća pravila:

- Koristi se `Parse` za zamenu `ParseFiles` jer `Parse` može da testira sadržaj direktno iz stringova, tako da nam nisu potrebne nikakve dodatne datoteke.
- Koristite `main` za svaki primer i nemojte koristiti `handler`.
- Koristi se `os.Stdout` za zamenu `http.ResponseWriter` jer `os.Stdout` takođe implementira `io.Writer` interfejs.

### Ubacivanje podataka u šablon

Upravo smo vam pokazali kako da analizirate i renderujete šablone. Hajde da odemo korak dalje i renderujemo podatke u naše šablone. Svaki šablon je objekat u programskom jeziku Go, pa kako da ubacimo polja u šablone?

#### Polja

U programu Go, svako polje koje želite da se prikaže unutar šablona treba da bude stavljeno unutar `{{}}`. `{{.}}` je skraćenica za trenutni objekat, što je slično njegovom pandanu u Javi ili C++. Ako želite da pristupite poljima trenutnog objekta, trebalo bi da koristite `{{.FieldName}}`. Obratite pažnju da se u šablonima može pristupiti samo izvezenim poljima. Evo primera:

```go
package main

import (
    "html/template"
    "os"
)

type Person struct {
    UserName string
}

func main() {
    t := template.New("fieldname example")
    t, _ = t.Parse("hello {{.UserName}}!")
    p := Person{UserName: "Astaxie"}
    t.Execute(os.Stdout, p)
}
```

Gornji primer se prikazuje "hello Astaxie" ispravno, ali ako malo izmenimo našu strukturu, pojavljuje se sledeća greška:

```go
type Person struct {
    UserName string
    email    string  // Field is not exported.
}

t, _ = t.Parse("hello {{.UserName}}! {{.email}}")
```

Ovaj deo koda neće biti kompajliran jer pokušavamo da pristupimo polju koje nije izvezeno. Međutim, ako pokušamo da koristimo polje koje ne postoji, Go jednostavno izbacuje prazan string umesto greške.

Ako štampate `{{.}}` u šablonu, Go izbacuje formatirani string ovog objekta, pozivajući funkciju `fmt` ispod haube.

#### Ugnežđena polja

Sada znamo kako da ispišemo polje. Šta ako je polje objekat i takođe ima svoja polja? Kako da ih sva ispišemo u jednoj petlji? Možemo koristiti `{{with …}}…{{end}}` i `{{range …}}{{end}}` upravo za tu svrhu.

`{% raw %} {{range}}{% endraw %}` baš kao `range` u Go-u.
`{% raw %} {{with}}{% endraw %}` vam omogućava da jednom napišete isto ime objekta i koristite ga kao skraćenicu ( slično kao `with` u VB-u ).

Više primera:

```go
package main

import (
    "html/template"
    "os"
)

type Friend struct {
    Fname string
}

type Person struct {
    UserName string
    Emails   []string
    Friends  []*Friend
}

func main() {
    f1 := Friend{Fname: "minux.ma"}
    f2 := Friend{Fname: "xushiwei"}
    t := template.New("fieldname example")
    t, _ = t.Parse(`hello {{.UserName}}!
            {{range .Emails}}
                an email {{.}}
            {{end}}
            {{with .Friends}}
            {{range .}}
                my friend name is {{.Fname}}
            {{end}}
            {{end}}
            `)
    p := Person{UserName: "Astaxie",
        Emails:  []string{"astaxie@beego.me", "astaxie@gmail.com"},
        Friends: []*Friend{&f1, &f2}}
    t.Execute(os.Stdout, p)
}
```

#### Uslovi

Ako treba da proverite uslove u šablonima, možete koristiti `if-else` sintaksu baš kao što biste to radili u redovnim Go programima. Ako je cevovod prazan, podrazumevana vrednost `if` je `false`. Sledeći primer pokazuje kako se koristi `if-else` u šablonima:

```go
package main

import (
    "os"
    "text/template"
)

func main() {
    tEmpty := template.New("template test")
    tEmpty = template.Must(tEmpty.Parse("Empty pipeline if demo: {{if ``}} will not be outputted. {{end}}\n"))
    tEmpty.Execute(os.Stdout, nil)

    tWithValue := template.New("template test")
    tWithValue = template.Must(tWithValue.Parse("Not empty pipeline if demo: {{if `anything`}} will be outputted. {{end}}\n"))
    tWithValue.Execute(os.Stdout, nil)

    tIfElse := template.New("template test")
    tIfElse = template.Must(tIfElse.Parse("if-else demo: {{if `anything`}} if part {{else}} else part.{{end}}\n"))
    tIfElse.Execute(os.Stdout, nil)
}
```

Kao što vidite, lako ga je koristiti `if-else` u šablonima.

> [!Note]
> Ne možete koristiti uslovne izraze u `if`, na primer `Mail=="astaxie@gmail.com"`. Prihvatljive
> su samo bulove vrednosti.

#### Cevovodi

Korisnici Unixa bi trebalo da budu upoznati sa `pipe` operatorom, kao što je `ls | grep "beego"`. Ova komanda filtrira datoteke i prikazuje samo one koje sadrže reč beego. Jedna stvar koja mi se sviđa kod Go šablona je to što podržavaju cevovode. Bilo šta u njima `{{}}` može biti podatak cevovoda. Imejl koji smo koristili gore može učiniti našu aplikaciju ranjivom na XSS napade. Kako možemo da rešimo ovaj problem korišćenjem cevovoda?

```html
{{. | html}}
```

Ovu metodu možemo koristiti da pretvorimo telo imejla u HTML. Veoma je slično pisanju Unix komande i pogodno je za upotrebu u šablonskim funkcijama.

#### Promenljive šablona

Ponekad nam je potrebno da koristimo lokalne promenljive u šablonima. Možemo ih koristiti sa ključnim rečima `with`, `range` i `if`, a njihov opseg je između ovih ključnih reči i `{{end}}`. Evo primera deklarisanja globalne promenljive:

```go
$variable := pipeline
```

Više primera:

```html
{{with $x := "output" | printf "%q"}}{{$x}}{{end}}
{{with $x := "output"}}{{printf "%q" $x}}{{end}}
{{with $x := "output"}}{{$x | printf "%q"}}{{end}}
```

#### Funkcije šablona

Go koristi `fmt` paket za formatiranje izlaza u šablonima, ali ponekad moramo da uradimo nešto drugo. Na primer, razmotrimo sledeći scenario: recimo da želimo da zamenimo `@` sa `at` u našoj imejl adresi, kao što je `astaxie at beego.me`. U ovom trenutku, moramo da napišemo prilagođenu funkciju.

Svaka šablonska funkcija ima jedinstveno ime i povezana je sa jednom funkcijom u vašem Go programu na sledeći način:

```go
type FuncMap map[string]interface{}
```

Pretpostavimo da imamo `emailDeal` šablonsku funkciju povezanu sa njenom `EmailDealWith` pandanom u našem Go programu. Možemo koristiti sledeći kod da registrujemo ovu funkciju:

```go
t = t.Funcs(template.FuncMap{"emailDeal": EmailDealWith})
```

`EmailDealWith` definicija:

```go
func EmailDealWith(args …interface{}) string
```

Primer:

```go
package main

import (
    "fmt"
    "html/template"
    "os"
    "strings"
)

type Friend struct {
    Fname string
}

type Person struct {
    UserName string
    Emails   []string
    Friends  []*Friend
}

func EmailDealWith(args ...interface{}) string {
    ok := false
    var s string
    if len(args) == 1 {
        s, ok = args[0].(string)
    }
    if !ok {
        s = fmt.Sprint(args...)
    }
    // find the @ symbol
    substrs := strings.Split(s, "@")
    if len(substrs) != 2 {
        return s
    }
    // replace the @ by " at "
    return (substrs[0] + " at " + substrs[1])
}

func main() {
    f1 := Friend{Fname: "minux.ma"}
    f2 := Friend{Fname: "xushiwei"}
    t := template.New("fieldname example")
    t = t.Funcs(template.FuncMap{"emailDeal": EmailDealWith})
    t, _ = t.Parse(`hello {{.UserName}}!
                {{range .Emails}}
                    an emails {{.|emailDeal}}
                {{end}}
                {{with .Friends}}
                {{range .}}
                    my friend name is {{.Fname}}
                {{end}}
                {{end}}
                `)
    p := Person{UserName: "Astaxie",
        Emails:  []string{"astaxie@beego.me", "astaxie@gmail.com"},
        Friends: []*Friend{&f1, &f2}}
    t.Execute(os.Stdout, p)
}
```

#### Must

Paket šablona ima funkciju koja se zove `Must` koja služi za validaciju šablona, kao što je podudaranje zagrada, komentara i promenljivih. Pogledajmo primer `Must`:

```go
package main

import (
    "fmt"
    "text/template"
)

func main() {
    tOk := template.New("first")
    template.Must(tOk.Parse(" some static text /* and a comment */"))
    fmt.Println("The first one parsed OK.")

    template.Must(template.New("second").Parse("some static text {{ .Name }}"))
    fmt.Println("The second one parsed OK.")

    fmt.Println("The next one ought to fail.")
    tErr := template.New("check parse error with Must")
    template.Must(tErr.Parse(" some static text {{ .Name }"))
}
```

Izlaz:

```sh
The first one parsed OK.
The second one parsed OK.
The next one ought to fail.
panic: template: check parse error with Must:1: unexpected "}" in command
```

### Ugnežđeni šabloni

Baš kao i u većini veb aplikacija, određeni delovi šablona mogu se ponovo koristiti u drugim šablonima, poput zaglavlja i podnožja bloga. Možemo deklarisati header, content i footer kao podšablone, i deklarisati ih u Go-u koristeći sledeću sintaksu:

```html
{{define "sub-template"}}content{{end}}
```

Podšablon se poziva korišćenjem sledeće sintakse:

```html
{{template "sub-template"}}
```

Evo kompletnog primera, pretpostavimo da imamo sledeće tri datoteke: header.tmpl, content.tmpli footer.tmplu fascikli templates, pročitaćemo fasciklu i sačuvati imena datoteka u nizu stringova, koji ćemo zatim koristiti za parsiranje datoteka.

Glavni šablon:

```html
{% raw %}
//header.tmpl
{{define "header"}}
<html>
<head>
    <title>Something here</title>
</head>
<body>
{{end}}

//content.tmpl
{{define "content"}}
{{template "header"}}
<h1>Nested here</h1>
<ul>
    <li>Nested usag</li>
    <li>Call template</li>
</ul>
{{template "footer"}}
{{end}}

//footer.tmpl
{{define "footer"}}
</body>
</html>
{{end}}

// When using subtemplating make sure that you have parsed each sub template file,
// otherwise the compiler wouldn't understand what to substitute when it 
// reads the {{template "header"}}

{% endraw %}
```

Kod:

```go
package main

import (
    "fmt"
    "os"
    "io/ioutil"
    "text/template"
    "strings"
)

var templates *template.Template

func main() {
    var allFiles []string
    files, err := ioutil.ReadDir("./templates")
    if err != nil {
        fmt.Println(err)
    }
    for _, file := range files {
        filename := file.Name()
        if strings.HasSuffix(filename, ".tmpl") {
            allFiles = append(allFiles, "./templates/"+filename)
        }
    }

    templates, err = template.ParseFiles(allFiles...) // parses all .tmpl files in the 'templates' folder

    s1 := templates.Lookup("header.tmpl")
    s1.ExecuteTemplate(os.Stdout, "header", nil)
    fmt.Println()
    s2 := templates.Lookup("content.tmpl")
    s2.ExecuteTemplate(os.Stdout, "content", nil)
    fmt.Println()
    s3 := templates.Lookup("footer.tmpl")
    s3.ExecuteTemplate(os.Stdout, "footer", nil)
    fmt.Println()
    s3.Execute(os.Stdout, nil)
}
```

Ovde možemo videti da `template.ParseFiles` analizira sve ugnežđene šablone u keš memoriju i da je
`{{define}}` svaki šablon definisan nezavisan jedan od drugog. Oni se čuvaju u nečemu poput mape, gde su imena šablona ključevi, a vrednosti tela šablona. Zatim možemo koristiti `ExecuteTemplate` da izvršimo odgovarajuće podšablone, tako da zaglavlje i podnožje budu nezavisni, a sadržaj ih sadrži oba. Imajte na umu da ako pokušamo da izvršimo `s1.Execute`, ništa se neće ispisati jer nema dostupnog podrazumevanog podšablona.

Kada ne želite da koristite `{{define}}`, onda možete jednostavno da kreirate tekstualnu datoteku sa imenom podšablona, na primer, `_head.tmpl` je podšablon koji ćete koristiti u celom projektu, a zatim da kreirate ovu datoteku u folderu šablona i koristite normalnu sintaksu. Keš pretrage je u osnovi kreiran tako da ne čitate datoteku svaki put kada pošaljete zahtev, jer ako to uradite, onda trošite mnogo resursa na čitanje datoteke koja se neće promeniti osim ako se kodna baza ne prepisuje. Nema smisla analizirati datoteke šablona tokom svakog HTTP GET zahteva, pa se koristi tehnika gde analiziramo datoteke jednom, a zatim vršimo `Lookup()`na kešu da bismo izvršili šablon kada nam je potreban za prikaz podataka.

Šabloni u jednom skupu se poznaju, ali ih morate analizirati za svaki pojedinačni skup.

Ponekad želite da kontekstualizujete šablone, na primer, imate `_head.html`, možda imate zaglavlje čiju vrednost morate da popunite na osnovu podataka koje učitavate, na primer, za menadžer liste zadataka možete imati tri kategorije pending, completed, deleted. Za ovo pretpostavimo da imate if naredbu kao što je ova.

```html
<title>{{if eq .Navigation "pending"}} Tasks
    {{ else if eq .Navigation "completed"}}Completed
    {{ else if eq .Navigation "deleted"}}Deleted
    {{ else if eq .Navigation "edit"}} Edit
    {{end}}
</title>
```

**Napomena**:  
Go šabloni prate poljsku notaciju prilikom poređenja, gde prvo navodite operator, zatim vrednost poređenja i vrednost sa kojom se upoređuje. Deo "else if" je prilično jednostavan.

Obično koristimo `{{ range }}` operator za prolazak kroz kontekstnu promenljivu koju prosleđujemo šablonu tokom izvršavanja ovako:

```go
//present in views package
context := db.GetTasks("pending") //true when you want non deleted notes
homeTemplate.Execute(w, context)
```

Kontekstni objekat dobijamo iz baze podataka kao strukturni objekat, definicija je sledeća:

```go
//Task is the struct used to identify tasks
type Task struct {
       Id      int

//Context is the struct passed to templates
type Context struct {

//present in database package

context = types.Context{Tasks: task, Navigation: status}

```

Koristimo niz zadataka i navigaciju u našim šablonima, videli smo kako koristimo navigaciju u šablonu, videćemo kako ćemo koristiti stvarni niz zadataka u našem šablonu.

Ovde `{{ if .Tasks }}` prvo proveravamo da li je polje "Tasks" našeg kontekstnog objekta, koje smo prosledili šablonu tokom izvršavanja, prazno ili ne. Ako nije prazno, onda ćemo proći kroz taj niz da bismo popunili naslov i sadržaj "Tasks". Donji primer je veoma važan kada je u pitanju petlja kroz niz u šablonu, počinjemo sa operatorom opsega, zatim možemo dati bilo kom članu te strukture kao "{{.Name}}", moja struktura "Tasks" ima naslov i sadržaj (obratite pažnju na velika T i C, to su izvezena imena i moraju biti napisana velikim slovom osim ako ne želite da ih učinite privatnim).

```html
{{ range .Tasks }}
    {{ .Title }}
    {{ .Content }}
{{ end }}
```

```html
    </span>

{{end}} {{else}}

    <p class="noteHeading">No Tasks here</p>

```

## 7.5 Datoteke

Datoteke su neophodni objekti na svakom računarskom uređaju. Neće vas iznenaditi da ih i veb aplikacije intenzivno koriste. U ovom odeljku ćemo naučiti kako da radimo sa datotekama u programskom jeziku Go.

### Direktorijumi

U programu Go, većina funkcija za rad sa datotekama nalazi se u `os` paketu. Evo nekih funkcija direktorijuma:

- **Mkdir**

  ```go
  func Mkdir(name string, perm FileMode) error
  ```
  
  Kreira direktorijum sa `name`. `Perm` su dozvole direktorijuma, tj. 0777.

- **MkdirAll**

  ```go
  func MkdirAll(path string, perm FileMode) error
  ```
  
  Kreira više direktorijuma prema `path`, kao što je "astaxie/test1/test2".

- Remove

  ```go
  func Remove(name string) error
  ```
  
  Uklanja direktorijum sa `name`. Vraća grešku ako nije direktorijum ili nije prazan.

- **RemoveAll**  

  ```go
  func RemoveAll(path string) error
  ```

Uklanja više direktorijuma u ​​skladu sa `path`. Direktorijumi neće biti obrisani ako `path` je jedna putanja.

Primer koda:

```go
package main

import (
    "fmt"
    "os"
)

func main() {
    os.Mkdir("astaxie", 0777)
    os.MkdirAll("astaxie/test1/test2", 0777)
    err := os.Remove("astaxie")
    if err != nil {
        fmt.Println(err)
    }
    os.RemoveAll("astaxie")
}
```

### Datoteke

#### Kreiranje i otvaranje datoteke

Postoje dve funkcije za kreiranje datoteka:

- **Create

  ```go
  func Create(name string) (file *File, err Error)
  ```
  
  Kreira datoteku imena `name` i vraća objekat `*File` datoteke koji se može čitati i pisati sa dozvolom 0666.

- **NewFile**

  ```go
  func NewFile(fd uintptr, name string) *File
  ```

  Kreira novu datoteku koja već postoji, ima hendl `fd` i vraća objekat datoteke `*File`.

Takođe postoje dve funkcije za otvaranje datoteka:

- **Open**

  ```go
  func Open(name string) (file *File, err Error)
  ```

  Otvara datoteku `name` samo za čitanje, pozivajući `OpenFile` ispod haube. Vraća objekat `*File`.

- **OpenFile**

  ```go
  func OpenFile(name string, flag int, perm uint32) (file *File, err Error)
  ```

  Otvara datoteku pod nazivom `name`. `flag` je režim otvaranja kao što je samo za čitanje, čitanje-pisanje itd. `Perm` su dozvole za datoteku. Vraća objekat datoteke `*File`.

#### Pisanje datoteka

Funkcije za pisanje datoteka:

- **Write**

  ```go
  func (file *File) Write(b []byte) (n int, err Error)
  ```

  Upisuje sadržaj `b` tipa isečka bajtova u datoteku `*File`.

- **WriteAt**

  ```go
  func (file *File) WriteAt(b []byte, off int64) (n int, err Error)
  ```

  Upisuje sadržaj isečka bajtova `b` na određenu poziciju `off` u datoteci `*File`.

- **WriteString**

  ```go
  func (file *File) WriteString(s string) (ret int, err Error)
  ```

  Upisuje string `s` u datoteku `*File`.

Primer koda:

```go
package main

import (
    "fmt"
    "os"
)

func main() {
    userFile := "astaxie.txt"
    fout, err := os.Create(userFile)        
    if err != nil {
        fmt.Println(userFile, err)
        return
    }
    defer fout.Close()
    for i := 0; i < 10; i++ {
        fout.WriteString("Just a test!\r\n")
        fout.Write([]byte("Just a test!\r\n"))
    }
}
```

#### Čitanje datoteka

Funkcije za čitanje datoteka:

- **Read**

  ```go
  func (file *File) Read(b []byte) (n int, err Error)
  ```
  
  Čita podatke iz `*File` u `b` isečak bajtova.

- **ReadAt**

  ```go
  func (file *File) ReadAt(b []byte, off int64) (n int, err Error)
  ```

  Čita podatke iz `*File` od pozicije `off` u isečak `b`. Vraća broj pročitanih bajtova.

Primer koda:

```go
package main

import (
    "fmt"
    "os"
)

func main() {
    userFile := "asatxie.txt"
    fl, err := os.Open(userFile)        
    if err != nil {
        fmt.Println(userFile, err)
        return
    }
    defer fl.Close()
    buf := make([]byte, 1024)
    for {
        n, _ := fl.Read(buf)
        if 0 == n {
            break
        }
        os.Stdout.Write(buf[:n])
    }
}
```

#### Brisanje datoteke

Go koristi istu funkciju za brisanje datoteka i direktorijuma:

- **Remove

  ```go
  func Remove(name string) Error
  ```

  Uklonja datoteku ili direktorijum pod nazivom name.

> [!Note]
> `name/` označava da je u pitanju direktorijum.

## 7.6 Stringovi

Na vebu, skoro sve što vidimo (uključujući korisničke unose, pristup bazi podataka itd.), predstavljeno je stringovima. Oni su veoma važan deo veb razvoja. U mnogim slučajevima, takođe nam je potrebno da delimo, spajamo, konvertujemo i na drugi način manipulišemo stringovima. U ovom odeljku ćemo predstaviti pakete `strings` i `strconv` iz standardne Go biblioteke.

### strings

Sledeće funkcije su iz `strings` paketa. Više detalja potražite u zvaničnoj dokumentaciji:

- **Contains**

  ```go
  func Contains(s, substr string) bool
  ```
  
  Proverava da li string `s` sadrži string `substr`, vraća bulovsku vrednost.
  
  ```go
  fmt.Println(strings.Contains("seafood", "foo"))
  fmt.Println(strings.Contains("seafood", "bar"))
  fmt.Println(strings.Contains("seafood", ""))
  fmt.Println(strings.Contains("", ""))
  ```
  
  Output:
  
  ```sh
  true
  false
  true
  true
  ```

- **Join**

  ```go
  func Join(a []string, sep string) string
  ```
  
  Kombinuje stringove iz isečka `a` pomoću separatora `sep`.
  
  ```go
  s := []string{"foo", "bar", "baz"}
  fmt.Println(strings.Join(s, ", "))
  //Output:foo, bar, baz
  ```

- **Index**

  ```go
  func Index(s, sep string) int
  ```

  Pronalazi indeks `sep` u stringu `s`, vraća `-1` ako nije pronađen.
  
  ```go
  fmt.Println(strings.Index("chicken", "ken"))
  fmt.Println(strings.Index("chicken", "dmr"))
  //Output:4
  //-1
  ```

- Repeat

  ```go
  func Repeat(s string, count int) string
  ```
  
  Ponovite string s count puta.
  
  ```go
  fmt.Println("ba" + strings.Repeat("na", 2))
  //Output:banana
  ```

- **Replace**

  ```go
  func Replace(s, old, new string, n int) string
  ```
  
  Zameni string `old` sa stringom `new` u stringu `s`. `n` je broj zamena. Ako je `n` manje od 0, zameni sve instance.
  
  ```go
  fmt.Println(strings.Replace("oink oink oink", "k", "ky", 2))
  fmt.Println(strings.Replace("oink oink oink", "oink", "moo", -1))
  //Output:oinky oinky oink
  //moo moo moo
  ```

- **Split**

  ```go
  funkcija Split(s, sep string) []string
  ```
  
  Podeli string `s` pomoću separatora `sep` na isečak.
  
  ```go
  fmt.Printf("%q\n", strings.Split("a,b,c", ","))
  fmt.Printf("%q\n", strings.Split("a man a plan a canal panama", "a "))
  fmt.Printf("%q\n", strings.Split(" xyz ", ""))
  fmt.Printf("%q\n", strings.Split("", "Bernardo O'Higgins"))
  //Output:["a" "b" "c"]
  //["" "man " "plan " "canal panama"]
  //[" " "x" "y" "z" " "]
  //[""]
  ```

- **Trim**

  ```go
  func Trim(s string, cutset string) string
  ```
  
  Uklanja `cutset` string iz stringa `s` ako je krajnje levo ili krajnje desno.
  
  ```go
  fmt.Printf("[%q]", strings.Trim(" !!! Achtung !!! ", "! "))
  // Output:["Achtung"]
  ```

- **Field**

```go
func Fields(s string) []string
```

Uklanja elemente razmaka i delite string sa razmakom na isečkom.

```go
fmt.Printf("Fields are: %q", strings.Fields("  foo bar  baz   "))
//Output:Fields are: ["foo" "bar" "baz"]
```

### strconv

Sledeće funkcije su iz strconv paketa. Kao i obično, pogledajte zvaničnu dokumentaciju za više detalja:

- **Append***

  Dodaj seriju, konvertuj podatke u string i dodaj trenutnom bajtnom isečku.
  
  ```go
  package main
  
  import (
      "fmt"
      "strconv"
  )
  
  func main() {
      str := make([]byte, 0, 100)
      str = strconv.AppendInt(str, 4567, 10)
      str = strconv.AppendBool(str, false)
      str = strconv.AppendQuote(str, "abcdefg")
      str = strconv.AppendQuoteRune(str, '单')
      fmt.Println(string(str))
  }
  ```

- **Format***  

  Formatiranje serija, konvertovanje drugih tipova podataka u string.
  
  ```go
  package main
  
  import (
      "fmt"
      "strconv"
  )
  
  func main() {
      a := strconv.FormatBool(false)
      b := strconv.FormatFloat(123.23, 'g', 12, 64)
      c := strconv.FormatInt(1234, 10)
      d := strconv.FormatUint(12345, 10)
      e := strconv.Itoa(1023)
      fmt.Println(a, b, c, d, e)
  }
  ```

- **Parse***

  Raščlanjivanje serija, konvertovanje stringova u druge tipove.
  
  ```go
  package main
  
  import (
      "fmt"
      "strconv"
  )
  
  func main() {
      a, err := strconv.ParseBool("false")
      if err != nil {
          fmt.Println(err)
      }
      b, err := strconv.ParseFloat("123.23", 64)
      if err != nil {
          fmt.Println(err)
      }
      c, err := strconv.ParseInt("1234", 10, 64)
      if err != nil {
          fmt.Println(err)
      }
      d, err := strconv.ParseUint("12345", 10, 64)
      if err != nil {
          fmt.Println(err)
      }
      e := strconv.Itoa(1023)
      fmt.Println(a, b, c, d, e)
  }
  ```
