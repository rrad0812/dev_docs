
# 5 Baze podataka

[Sadržaj](_00-sr.md)

Za veb programere, baza podataka je srž veb razvoja. Možete sačuvati skoro sve u bazi podataka i upitivati ili ažurirati podatke unutar nje, kao što su korisničke informacije, proizvodi ili novinski članci.

Go ne pruža nikakve drajvere za baze podataka, ali ima definisan interfejs drajvera u database/sqlpaketu. Ljudi mogu da razvijaju drajvere za baze podataka na osnovu tog interfejsa. U odeljku 5.1 ćemo govoriti o dizajnu interfejsa drajvera za baze podataka u Go-u. U odeljcima 5.2 do 5.4, predstaviću vam neke SQL drajvere za baze podataka. U odeljku 5.5, predstaviću ORM koji sam razvio, a koji je zasnovan na database/sqlstandardu interfejsa. Kompatibilan je sa većinom drajvera koji su implementirali database/sqlinterfejs i olakšava pristup bazama podataka idiomatski u Go-u.

NoSQL je poslednjih godina bila vruća tema. Sve više veb-sajtova se odlučuje da koristi NoSQL baze podataka kao svoju glavnu bazu podataka umesto samo za keširanje. U odeljku 5.6 ću vam predstaviti dve NoSQL baze podataka, MongoDB i Redis.

## 5.1 Interfejs database/SQL

Go ne pruža nikakve zvanične drajvere za baze podataka, za razliku od drugih jezika poput PHP-a koji ih pružaju. Međutim, ima neke standarde interfejsa drajvera za baze podataka koje programeri mogu da koriste za razvoj drajvera za baze podataka. Prednost je u tome što ako je vaš kod razvijen u skladu sa ovim standardima interfejsa, nećete morati da menjate nikakav kod ako se vaša baza podataka promeni. Hajde da vidimo koji su ovi standardi interfejsa za baze podataka.

### sql.Registar

Ova funkcija se nalazi u `database/sql` paketu za registrovanje drajvera baze podataka kada koristite drajvere baze podataka trećih strana. Svi oni treba da pozovu `Register(name string, driver driver.Driver)` funkciju u `init()` da bi se registrovali.

Hajde da pogledamo odgovarajući kod drajvera za sqlite3 i mymysql:

```go
//https://github.com/mattn/go-sqlite3 driver
func init() {
    sql.Register("sqlite3", &SQLiteDriver{})
}
```

```go
//https://github.com/mikespook/mymysql driver
// Driver automatically registered in database/sql
var d = Driver{proto: "tcp", raddr: "127.0.0.1:3306"}
func init() {
    Register("SET NAMES utf8")
    sql.Register("mymysql", &d)
}
```

Vidimo da svi drajveri baza podataka trećih strana implementiraju ovu funkciju da bi se registrovali, a Go koristi mapu da bi sačuvao korisničke drajvere unutar `database/sql`.

```go
var drivers = make(map[string]driver.Driver)
drivers[name] = driver
```

Stoga, ova funkcija registracije može da registruje onoliko drajvera koliko vam je potrebno, svaki sa različitim imenima.

Uvek vidimo sledeći kod kada koristimo drajvere treće strane:

```go
import (
    "database/sql"
    _ "github.com/mattn/go-sqlite3"
)
```

Ovde, donja crtica (poznata i kao „blank“) `_` može biti prilično zbunjujuća za mnoge početnike, ali ovo je odlična funkcija u Gou. Već znamo da se ovaj identifikator donje crte koristi za odbacivanje vrednosti iz vraćanja funkcija, a takođe i da morate koristiti sve pakete koje ste uvezli u svom kodu u Gou. Dakle, kada se blank koristi sa importom, to znači da će da izvrši funkciju `init()` tog paketa bez direktnog korišćenja, što je savršeno za slučaj upotrebe registrovanja drajvera baze podataka.

### driver.Driver

Driver je interfejs koji sadrži `Open(name string)` metodu koja vraća `Conn` interfejs.

```go
type Driver interface {
    Open(name string) (Conn, error)
}
```

Ovo je jednokratna konverzacija, što znači da se može koristiti samo jednom po gorutini. Sledeći kod će izazvati greške:

```go
...
go goroutineA (Conn)  // query
go goroutineB (Conn)  // insert
...
```

Pošto Go nema pojma koja gorutina obavlja koju operaciju, operacija upita može dobiti rezultat operacije umetanja i obrnuto.

Svi drajveri trećih strana treba da imaju ovu funkciju imena `Conn` i vraćanje ispravnih rezultata.

### driver.Conn

Ovo je interfejs za povezivanje sa bazom podataka i sa nekim metodama, i kao što sam rekao gore, isti `Conn` se može koristiti samo jednom po goroutine.

```go
type Conn interface {
    Prepare(query string) (Stmt, error)
    Close() error
    Begin() (Tx, error)
}
```

- `Prepare` vraća status pripreme odgovarajućih SQL komandi za upite i brisanje itd.
- `Close` zatvara trenutnu vezu i čisti resurse. Većina drajvera trećih strana implementira neku vrstu pula konekcija, tako da ne morate keširati konekcije što može prouzrokovati neočekivane greške.
- `Begin` vraća `Tx` koji predstavlja identifikator transakcije. Možete ga koristiti za upite, ažuriranje, vraćanje transakcija itd.

### driver.Stmt

Ovo je status spremnosti koji odgovara `Conn`-u, tako da se može koristiti samo jednom po gorutini (kao što je slučaj sa Conn-om).

```go
type Stmt interface {
    Close() error
    NumInput() int
    Exec(args []Value) (Result, error)
    Query(args []Value) (Rows, error)
}
```

- `Close` zatvara trenutnu vezu, ali i dalje vraća podatke reda ako izvršava operaciju upita.
- `NumInput` vraća broj obaveznih argumenata. Drajveri baze podataka treba da provere argumente pozivaoca kada je rezultat veći od 0, a vraća -1 kada drajveri baze podataka ne poznaju nijedan obavezni argument.
- `Exec` izvršava update/insertSQL komande pripremljene u `Prepare`, vraća `Result`.
- `Query` izvršava select SQL komandu pripremljenu u `Prepare`, vraća podatke Rows.

### driver.Tx

Generalno, deskriptori transakcija imaju samo metode `Commit` i `Rollback`, a drajveri baze podataka treba da implementiraju samo ove dve metode.

```go
type Tx interface {
    Commit() error
    Rollback() error
}
```

### driver.Execer

Ovo je opcioni interfejs.

```go
type Execer interface {
    Exec(query string, args []Value) (Result, error)
}
```

Ako drajver ne implementira ovaj interfejs, kada pozovete `DB.Exec`, on će automatski pozvati `Prepare`, a zatim vratiti `Stmt`. Nakon toga izvršava `Exec` metodu `Stmt`, a zatim zatvara `Stmt`.

### diver.Result

Ovo je interfejs za rezultate update/insert operacija.

```go
type Result interface {
    LastInsertId() (int64, error)
    RowsAffected() (int64, error)
}
```

- `LastInsertId` vraća automatski uvećani ID broj nakon operacije umetanja u bazu podataka.
- `RowsAffected` vraća redove na koje su uticale operacije upita.

### driver.Rows

Ovo je interfejs za rezultat operacije upita.

```go
type Rows interface {
    Columns() []string
    Close() error
    Next(dest []Value) error
}
```

- `Columns` vraća informacije o poljima tabela baze podataka. Isečak ima jedan-na-jedan odnos samo sa poljima SQL upita i ne vraća sva polja te tabele baze podataka.
- `Close` zatvara iterator redova.
- `Next` vraća sledeće podatke i dodeljuje ih u `dest`, konvertujući sve stringove u nizove bajtova, i daje grešku `io.EOF` ako više nema dostupnih podataka.

### driver.RowsAffected

Ovo je alias za `int64`, ali implementira `Result` interfejs.

```go
type RowsAffected int64
func (RowsAffected) LastInsertId() (int64, error)
func (v RowsAffected) RowsAffected() (int64, error)
```

### driver.Value

Ovo je prazan interfejs koji može da sadrži bilo koju vrstu podataka.

```go
type Value interface{}
```

Vrednost mora biti nešto na čemu drajveri mogu da rade ili jednaka nuli, tako da bi trebalo da bude jednog od sledećih tipova:

```go
int64
float64
bool
[]byte
string   // [*] Except Rows.Next which cannot return string
time.Time
```

### driver.ValueConverter

Ovo definiše interfejs za pretvaranje normalnih vrednosti u driver.Value.

```go
type ValueConverter interface {
    ConvertValue(v interface{}) (Value, error)
}
```

Ovaj interfejs se često koristi u drajverima baza podataka i ima mnogo korisnih funkcija:

- Konvertuje `driver.Value` u odgovarajući tip polja baze podataka, na primer, konvertuje `int64` u `uint16`.
- Konvertuje rezultate upita baze podataka u vrednost tipa `driver.Value`.
- Konvertuje `driver.Value` u korisnički definisanu vrednost u `scan` funkciji.

### driver.Valuer

Ovo definiše interfejs za vraćanje vrednosti `driver.Value`.

```go
type Valuer interface {
    Value() (Value, error)
}
```

Mnogi tipovi implementiraju ovaj interfejs za konverziju između `driver.Value` i samog sebe.

U ovom trenutku, trebalo bi da znate malo o razvoju drajvera za baze podataka u Go jeziku. Kada budete mogli da implementirate interfejse za operacije poput dodavanja, brisanja, ažuriranja itd., preostaje samo nekoliko problema vezanih za komunikaciju sa određenim bazama podataka.

### database/SQL

database/sql definiše još više metoda višeg nivoa pored database/sql/driver za praktičnije operacije sa bazom podataka i predlaže da implementirate pul konekcija.

```go
type DB struct {
    driver   driver.Driver
    dsn      string
    mu       sync.Mutex // protects freeConn and closed
    freeConn []driver.Conn
    closed   bool
}
```

Kao što vidite, `Open` funkcija vraća bazu podataka koja ima `freeConn`, i to je jednostavan pul konekcija. Njena implementacija je veoma jednostavna i ružna. Koristi se `defer db.putConn(ci, err)` u funkciji `Db.prepare` da bi se konekcija stavila u pul konekcija. Svaki put kada pozovete funkciju `Conn`, ona proverava dužinu `freeConn`. Ako je veća od 0, to znači da postoji veza koja se može ponovo koristiti i direktno vam se vraća. U suprotnom, kreira novu konekciju i vraća se.

## 5.2 MySQL

LAMP stek je poslednjih godina bio veoma popularan na internetu, a M u LAMP-u označava MySQL. MySQL je poznat jer je otvorenog koda i jednostavan za korišćenje. Kao takav, postao je de fakto baza podataka u pozadinskim sistemima mnogih veb-sajtova.

### MySQL drajveri

Postoji nekoliko drajvera koji podržavaju MySQL u Go-u. Neki od njih implementiraju interfejs `database/sql`, dok drugi koriste sopstvene standarde interfejsa.

<https://github.com/go-sql-driver/mysql> podržava `database/sql`, napisan u čistom Go-u.
<https://github.com/ziutek/mymysql> podržava `database/sql` korisnički definisane interfejse, napisane u čistom Go jeziku.

U sledećim primerima ću koristiti prvi drajver (koristim ga i u svojim ličnim projektima), a preporučujem da ga koristite i vi iz sledećih razloga:

- To je novi drajver baze podataka i podržava više funkcija.
- U potpunosti podržava `database/sql` standarde interfejsa.
- Podržava dugotrajne veze koje se održavaju u realnom vremenu sa bezbednošću niti.

### Primer za MySQL

U sledećim odeljcima, koristiću istu strukturu tabele baze podataka za različite baze podataka, a zatim ću kreirati SQL na sledeći način:

```sql
CREATE TABLE `userinfo` (
    `uid` INT(10) NOT NULL AUTO_INCREMENT,
    `username` VARCHAR(64) NULL DEFAULT NULL,
    `department` VARCHAR(64) NULL DEFAULT NULL,
    `created` DATE NULL DEFAULT NULL,
    PRIMARY KEY (`uid`)
);
```

Sledeći primer pokazuje kako se radi sa bazom podataka na osnovu `database/sql` standarda interfejsa.

```go
package main

import (
    _ "github.com/go-sql-driver/mysql"
    "database/sql"
    "fmt"
)

func main() {
    db, err := sql.Open("mysql", "astaxie:astaxie@/test?charset=utf8")
    checkErr(err)

    // insert
    stmt, err := db.Prepare("INSERT userinfo SET username=?,department=?,created=?")
    checkErr(err)

    res, err := stmt.Exec("astaxie", "研发部门", "2012-12-09")
    checkErr(err)

    id, err := res.LastInsertId()
    checkErr(err)
    fmt.Println(id)

// update
    stmt, err = db.Prepare("update userinfo set username=? where uid=?")
    checkErr(err)

    res, err = stmt.Exec("astaxieupdate", id)
    checkErr(err)

    affect, err := res.RowsAffected()
    checkErr(err)
    fmt.Println(affect)

    // query
    rows, err := db.Query("SELECT * FROM userinfo")
    checkErr(err)

    for rows.Next() {
        var uid int
        var username string
        var department string
        var created string
        
        err = rows.Scan(&uid, &username, &department, &created)
        checkErr(err)

        fmt.Println(uid)
        fmt.Println(username)
        fmt.Println(department)
        fmt.Println(created)
    }
    
    rows.Close()

    // delete
    stmt, err = db.Prepare("delete from userinfo where uid=?")
    checkErr(err)

    res, err = stmt.Exec(id)
    checkErr(err)

    affect, err = res.RowsAffected()
    checkErr(err)
    fmt.Println(affect)

    db.Close()
}

func checkErr(err error) {
    if err != nil {
        panic(err)
    }
}
```

Dozvolite mi da ovde objasnim nekoliko važnih funkcija:

- `sql.Open()` otvara registrovani drajver baze podataka. Go-MySQL-Driver je ovde registrovao "mysql" drajver. Drugi argument je DSN (naziv izvora podataka) koji definiše informacije koje se odnose na vezu sa bazom podataka. Podržava sledeće formate:

  - user@unix(/path/to/socket)/dbname?charset=utf8
  - user:password@tcp(localhost:5555)/dbname?charset=utf8
  - user:password@/dbname
  - user:password@tcp([de:ad:be:ef::ca:fe]:80)/dbname

- `db.Prepare()` vraća SQL operaciju koja će biti izvršena. Takođe vraća status izvršenja nakon izvršavanja SQL-a.

- `db.Query()` izvršava SQL i vraća rezultat tipa `Rows`.

- `stmt.Exec()` izvršava SQL koji je pripremljen i sačuvan u Stmt.

Imajte na umu da koristimo format `?` za prosleđivanje argumenata. Ovo je neophodno za sprečavanje SQL injekcija napada.

## 5.3 SQLite

SQLite je ugrađena relaciona baza podataka otvorenog koda. Ima samostalni mehanizam baze podataka koji ne zahteva nikakvu konfiguraciju i podržava transakcije. Njegove karakteristike su visoka prenosivost, jednostavan za korišćenje, kompaktan, efikasan i pouzdan. U većini slučajeva, potrebna vam je samo binarna datoteka SQLite-a da biste kreirali, povezali i upravljali bazom podataka. Ako tražite rešenje za ugrađenu bazu podataka, SQLite vredi razmotriti. Može se reći da je SQLite verzija Access-a otvorenog koda.

### SQLite drajveri

Postoji mnogo drajvera za baze podataka za SQLite u Go-u, ali mnogi od njih ne podržavaju `database/sql` standarde interfejsa.

<https://github.com/mattn/go-sqlite3> podržava `database/sql`, zasnovano na cgo.
<https://github.com/feyeleanor/gosqlite3> ne podržava `database/sql`, zasnovano na cgo.
<https://github.com/phf/go-sqlite3> ne podržava `database/sql`, zasnovano na cgo.

Prvi drajver je jedini koji podržava `database/sql` standard interfejsa u svom SQLite drajveru, tako da ga koristim u svojim projektima - olakšaće migraciju mog koda u budućnosti ako bude potrebno.

### Primer za SQLite3

Kreiramo sledeći SQL:

```sql
CREATE TABLE `userinfo` (
    `uid` INTEGER PRIMARY KEY AUTOINCREMENT,
    `username` VARCHAR(64) NULL,
    `department` VARCHAR(64) NULL,
    `created` DATE NULL
);
```

Primer:

```go
package main

import (
    "database/sql"
    "fmt"
    "time"
    _ "github.com/mattn/go-sqlite3"
)

func main() {
    db, err := sql.Open("sqlite3", "./foo.db")
    checkErr(err)

    // insert
    stmt, err := db.Prepare("INSERT INTO userinfo(username, department, created) values(?,?,?)")
    checkErr(err)

    res, err := stmt.Exec("astaxie", "研发部门", "2012-12-09")
    checkErr(err)

    id, err := res.LastInsertId()
    checkErr(err)
    fmt.Println(id)

// update
    stmt, err = db.Prepare("update userinfo set username=? where uid=?")
    checkErr(err)

    res, err = stmt.Exec("astaxieupdate", id)
    checkErr(err)

    affect, err := res.RowsAffected()
    checkErr(err)
    fmt.Println(affect)

    // query
    rows, err := db.Query("SELECT * FROM userinfo")
    checkErr(err)
    var uid int
    var username string
    var department string
    var created time.Time
    
    for rows.Next() {
        err = rows.Scan(&uid, &username, &department, &created)
        checkErr(err)
        fmt.Println(uid)
        fmt.Println(username)
        fmt.Println(department)
        fmt.Println(created)
    }
    
    rows.Close() //good habit to close
    
    // delete
    stmt, err = db.Prepare("delete from userinfo where uid=?")
    checkErr(err)

    res, err = stmt.Exec(id)
    checkErr(err)

    affect, err = res.RowsAffected()
    checkErr(err)
    fmt.Println(affect)

    db.Close()
}

func checkErr(err error) {
    if err != nil {
        panic(err)
    }
}
```

Možda ste primetili da je kod skoro isti kao u prethodnom odeljku, i da smo samo promenili ime registrovanog drajvera i pozvali `sql.Open` povezivanje sa SQLite-om na drugačiji način.

Imajte na umu da ponekad ne možete koristiti `for` izjavu jer nemate više od jednog reda, tada možete koristiti `if` izjavu

```go
if rows.Next() {
    err = rows.Scan(&uid, &username, &department, &created)
    checkErr(err)

    fmt.Println(uid)
    fmt.Println(username)
    fmt.Println(department)
    fmt.Println(created)
}
```

Takođe morate da uradite `rows.Next()`, bez korišćenja toga ne možete da preuzmete podatke u `Scan` funkciji.

### Transakcije

Gore navedeni primer pokazuje kako preuzimate podatke iz baze podataka, ali kada želite da napišete veb aplikaciju, onda neće biti potrebno samo preuzeti podatke iz baze podataka, već će biti potrebno i upisati podatke u nju. U tu svrhu, trebalo bi da koristite transakcije jer iz različitih razloga, kao što je postojanje više go rutina koje pristupaju bazi podataka, baza podataka može biti zaključana. Ovo je nepoželjno u vašoj veb aplikaciji i upotreba transakcija je efikasna u osiguravanju da aktivnosti vaše baze podataka prođu ili potpuno ne uspeju u zavisnosti od okolnosti. Jasno je da korišćenje transakcija može sprečiti da mnoge stvari krenu pogrešno sa veb aplikacijom.

```go
trashSQL, err := database.Prepare("update task set is_deleted='Y',last_modified_at=datetime() where id=?")
if err != nil {
    fmt.Println(err)
}
tx, err := database.Begin()
if err != nil {
    fmt.Println(err)
}
_, err = tx.Stmt(trashSQL).Exec(id)
if err != nil {
    fmt.Println("doing rollback")
    tx.Rollback()
} else {
    tx.Commit()
}
```

Kao što je jasno iz gornjeg bloka koda, prvo pripremate naredbu, nakon čega je izvršavate, u zavisnosti od rezultata tog izvršavanja, a zatim je ili vraćate unazad (Rollback) ili je potvrđujete (Commit).

Kao poslednja napomena o ovom odeljku, dostupan je koristan alat za upravljanje SQLite-om: <http://sqlitebrowser.org>

## 5.4 PostgreSQL

PostgreSQL je objektno-relacioni sistem za upravljanje bazama podataka dostupan za mnoge platforme, uključujući Linux, FreeBSD, Solaris, Microsoft Windows i Mac OS X. Objavljen je pod licencom u stilu MIT-a, te je stoga besplatan softver otvorenog koda. Veći je od MySQL-a jer je dizajniran za upotrebu u preduzećima kao alternativa Oracle-u. PostgreSQL je dobar izbor za projekte tipa preduzeća.

### PostgreSQL drajveri

Postoji mnogo drajvera za baze podataka dostupnih za PostgreSQL. Evo tri primera:

<https://github.com/lib/pq> podržava `database/sql`, napisan u čistom Go-u.
<https://github.com/jbarham/gopgsqldriver> podržava `database/sql`, napisan u čistom Go-u.
<https://github.com/lxn/go-pgsql> podržava `database/sql`, napisan u čistom Go-u.
U primerima koji slede koristiću prvi.

### Primer za PostgreSQL

Kreiramo sledeći SQL:

```sql
CREATE TABLE userinfo
(
    uid serial NOT NULL,
    username character varying(100) NOT NULL,
    department character varying(500) NOT NULL,
    Created date,
    CONSTRAINT userinfo_pkey PRIMARY KEY (uid)
)
WITH (OIDS=FALSE);
```

Primer:

```go
package main

import (
    "database/sql"
    "fmt"
    _ "github.com/lib/pq"
    "time"
)
const (
    DB_USER     = "postgres"
    DB_PASSWORD = "postgres"
    DB_NAME     = "test"
)
func main() {
    dbinfo := fmt.Sprintf("user=%s password=%s dbname=%s sslmode=disable",
        DB_USER, DB_PASSWORD, DB_NAME)
    db, err := sql.Open("postgres", dbinfo)
    checkErr(err)
    defer db.Close()

    fmt.Println("# Inserting values")

    var lastInsertId int
    err = db.QueryRow("INSERT INTO userinfo(username,department,created) VALUES($1,$2,$3) returning uid;", "astaxie", "研发部门", "2012-12-09").Scan(&lastInsertId)
    checkErr(err)
    fmt.Println("last inserted id =", lastInsertId)
    
    fmt.Println("# Updating")

    stmt, err := db.Prepare("update userinfo set username=$1 where uid=$2")
    checkErr(err)
    res, err := stmt.Exec("astaxieupdate", lastInsertId)
    checkErr(err)
    affect, err := res.RowsAffected()
    checkErr(err)
    fmt.Println(affect, "rows changed")
    
    fmt.Println("# Querying")

    rows, err := db.Query("SELECT * FROM userinfo")
    checkErr(err)
    for rows.Next() {
        var uid int
        var username string
        var department string
        var created time.Time
        err = rows.Scan(&uid, &username, &department, &created)
        checkErr(err)
        fmt.Println("uid | username | department | created ")
        fmt.Printf("%3v | %8v | %6v | %6v\n", uid, username, department, created)
    }
    
    fmt.Println("# Deleting")

    stmt, err = db.Prepare("delete from userinfo where uid=$1")
    checkErr(err)
    res, err = stmt.Exec(lastInsertId)
    checkErr(err)
    affect, err = res.RowsAffected()
    checkErr(err)
    fmt.Println(affect, "rows changed")
}
func checkErr(err error) {
    if err != nil {
        panic(err)
    }
}
```

Imajte na umu da PostgreSQL koristi `$1`, `$2` format umesto koji `?` koristi MySQL, i ima drugačiji DSN format u `sql.Open`. Još jedna stvar je da PostgreSQL drajver ne podržava `sql.Result.LastInsertId()`. Dakle, umesto ovoga,

```go
stmt, err := db.Prepare("INSERT INTO userinfo(username,departname,created) VALUES($1,$2,$3);")
res, err := stmt.Exec("astaxie", "研发部门", "2012-12-09")
fmt.Println(res.LastInsertId())
```

Koristite `db.QueryRow()` i `Scan()` da biste dobili vrednost za poslednji umetnuti ID.

```go
err = db.QueryRow("INSERT INTO TABLE_NAME values($1) returning uid;",    VALUE1").Scan(&lastInsertId)
fmt.Println(lastInsertId)
```

## 5.5 Razvijanje ORM zasnovanog na beedb-u

> [!Note]
> Projekat `beedb` se više ne održava, ali kod je i dalje tu.

`beedb` je ORM (maper objekata i relacije) koji sam ja razvio u Go-u. Koristi idiomatski Go za rad sa bazama podataka, implementirajući mapiranje struktura u bazu podataka i deluje kao lagani Go ORM okvir. Svrha razvoja ovog ORM-a nije samo da pomogne ljudima da nauče kako da pišu ORM, već i da pronađe dobru ravnotežu između funkcionalnosti i performansi kada je u pitanju perzistentnost podataka.

`beedb` je projekat otvorenog koda koji podržava osnovne ORM funkcionalnosti, ali ne podržava upite za asocijaciju.

Pošto `beedb` podržava `database/sql` standarde interfejsa, svaki drajver koji implementira ovaj interfejs može se koristiti sa `beedb`-om. Testirao sam sledeće drajvere:

- MySQL: github.com/go-mysql-driver/mysql
- PostgreSQL: github.com/lib/pq
- SQLite: github.com/mattn/go-sqlite3
- MySQL: github.com/ziutek/mymysql/godrv
- MS ADODB: github.com/mattn/go-adodb
- Oracle: github.com/mattn/go-oci8
- ODBC: bitbucket.org/miquella/mgodbc

### Instalacija

Možete koristiti `go get` za lokalnu instalaciju `beedb`-a.

```SH
go get github.com/astaxie/beedb
```

### Inicijalizacija

Prvo, morate da uvezete sve potrebne pakete:

```GO
import (
    "database/sql"
    "github.com/astaxie/beedb"
    _ "github.com/ziutek/mymysql/godrv"
)
```

Zatim treba da otvorite vezu sa bazom podataka i kreirate `beedb objekat` (MySQL u ovom primeru):

```go
db, err := sql.Open("mymysql", "test/xiemengjun/123456")
if err != nil {
    panic(err)
}
orm := beedb.New(db)
```

`beedb.New()` zapravo ima dva argumenta. Prvi je objekat baze podataka, a drugi služi za označavanje koji mehanizam baze podataka koristite. Ako koristite MySQL ili SQLite, možete jednostavno preskočiti drugi argument.

U suprotnom, ovaj argument mora biti naveden. Na primer, u slučaju SQLServer-a:

```go
orm = beedb.New(db, "mssql")
```

PostgreSQL:

```go
orm = beedb.New(db, "pg")
```

`beedb` podržava debagovanje. Koristite sledeći kod da biste ga omogućili:

```go
beedb.OnDebug=true
```

Zatim, imamo strukturu za "Userinfo" tabelu baze podataka koju smo koristili u prethodnim odeljcima.

```go
type Userinfo struct {
    Uid     int `PK`    // if the primary key is not id, you need to add tag `PK` 
                        // for your customized primary key.
    Username    string
    Departname  string
    Created     time.Time
}
```

Imajte na umu da `beedb` automatski konvertuje imena napisana malim slovom camelcase u mala slova snake. Na primer, ako imamo "UserInfo" kao ime strukture, `beedb` će ga konvertovati u "user_info" u bazi podataka. Isto pravilo važi i za imena polja strukture.

### Umetanje podatka

Sledeći primer vam pokazuje kako da koristite beedb za čuvanje strukture, umesto korišćenja sirovih SQL komandi. Koristimo beedb metodu Save da primenimo promenu.

```go
var saveone Userinfo
saveone.Username = "Test Add User"
saveone.Departname = "Test Add Departname"
saveone.Created = time.Now()
orm.Save(&saveone)
```

Možete proveriti "saveone.Uid" nakon što je zapis umetnut; njegova vrednost je samoinkrementirani ID, o čemu se brine metoda „Save“ umesto vas.

`beedb` pruža još jedan način umetanja podataka; to je preko Go-ovog tipa mape.

```go
add := make(map[string]interface{})
add["username"] = "astaxie"
add["departname"] = "cloud develop"
add["created"] = "2012-12-02"
orm.SetTable("userinfo").Insert(add)
```

Umetanje više podataka:

```go
addslice := make([]map[string]interface{}, 10)
add:=make(map[string]interface{})
add2:=make(map[string]interface{})
add["username"] = "astaxie"
add["departname"] = "cloud develop"
add["created"] = "2012-12-02"
add2["username"] = "astaxie2"
add2["departname"] = "cloud develop2"
add2["created"] = "2012-12-02"
addslice = append(addslice, add, add2)
orm.SetTable("userinfo").InsertBatch(addslice)
```

Gore prikazana metoda je slična lančanom upitu, sa kojim bi trebalo da budete upoznati ako ste ikada koristili jquery. Vraća originalni ORM objekat nakon poziva, a zatim nastavlja da obavlja druge zadatke.

Metoda `SetTable` govori ORM-u da želimo da ubacimo naše podatke u "userinfo" tabelu.

### Ažuriranje podataka

Nastavimo rad sa gornjim primerom da bismo videli kako da ažuriramo podatke. Sada kada imamo primarni ključ od saveone(Uid), beedb izvršava operaciju ažuriranja umesto umetanja novog zapisa.

```go
saveone.Username = "Update Username"
saveone.Departname = "Update Departname"
saveone.Created = time.Now()
orm.Save(&saveone)  // update
```

Kao i ranije, možete koristiti i mapu za ažuriranje podataka:

```go
t := make(map[string]interface{})
t["username"] = "astaxie"
orm.SetTable("userinfo").SetPK("uid").Where(2).Update(t)
```

Dozvolite mi da objasnim neke od gore navedenih metoda:

- `SetPK()`  - govori ORM-u da je uid primarni ključ zapisa u "userinfo" tabeli.
- `Where()`  - postavlja uslove i podržava više argumenata. Ako je prvi argument ceo broj, to je
               skraćenica za `Where("<primary key>=?", <value>)`.
- `Update()` - metoda prihvata mapu i ažurira bazu podataka.

### Upiti

`beedb` interfejs za upite je veoma fleksibilan. Pogledajmo nekoliko primera:

**Upit po primarnom ključu**:

```go
var user Userinfo
// Where accepts two arguments, supports integers
orm.Where("uid=?", 27).Find(&user)
```

**Skraćena verzija upita po primarnom ključu**:

```go
var user2 Userinfo
orm.Where(3).Find(&user2) // short form that omits primary key
```

**Upiti po ostalim uslovima**:

```go
var user3 Userinfo
// Where two arguments are accepted, with support for char type.
orm.Where("name = ?", "john").Find(&user3)
```

**Upiti po složenijim uslovima**:

```go
var user4 Userinfo
// Where three arguments are accepted
orm.Where("name = ? and age < ?", "john", 88).Find(&user4)
```

Primeri za dobijanje više zapisa:

**Upit pod uslovom id > 3 za 10 zapisa koji počinju sa 20 zapisom**:

```go
var allusers []Userinfo
err := orm.Where("id > ?", "3").Limit(10,20).FindAll(&allusers)
```

**Upit pod uslovom id > 3, za 10 zapisa, od 1 zapisa**:

```go
var tenusers []Userinfo
err := orm.Where("id > ?", "3").Limit(10).FindAll(&tenusers)
```

**Preuzmi sve zapise**:

```go
var everyone []Userinfo
err := orm.OrderBy("uid desc, username asc").FindAll(&everyone)
```

Kao što vidite, metoda `Limit` služi za ograničavanje broja rezultata.

- `Limit()` - podržava dva argumenta: broj rezultata i početnu poziciju. 0 je podrazumevana vrednost početne pozicije.

- `OrderBy()` služi za uređivanje rezultata. Argument je uslov uređivanja.

Svi primeri ovde su jednostavno mapiranje zapisa u strukture. Takođe možete jednostavno da unesete podatke u mapu na sledeći način:

```go
a, _ := orm.SetTable("userinfo").SetPK("uid").Where(2).Select("uid,username").FindMap()

- `Select()` govori beedb-u koliko polja želite da dobijete iz tabele baze podataka. Ako nije navedeno, sva polja se vraćaju podrazumevano.

- `FindMap()` vraća []map[string][]bytetip, tako da morate sami da konvertujete u druge tipove.
```

### Brisanje podataka

`beedb` pruža bogate metode za brisanje podataka.

**Brisanje jednog zapisa**:

```go
// saveone is the one in above example.
orm.Delete(&saveone)
```

**Brisanje više zapisa**:

```go
// alluser is the slice which gets multiple records.
orm.DeleteAll(&alluser)
```

**Brisanje zapisa pomoću SQL-a**:

```go
orm.SetTable("userinfo").Where("uid>?", 3).DeleteRow()
```

### Upiti asocijativni

`beedb` ne podržava spajanje struktura. Međutim, pošto je nekim aplikacijama potrebna ova funkcija, evo implementacije:

```go
a, _ := orm.SetTable("userinfo").Join("LEFT", "userdetail", "userinfo.uid=userdetail.uid")
    .Where("userinfo.uid=?", 1).Select("userinfo.uid,userinfo.username,userdetail.profile").FindMap()
```

Vidimo novu metodu koja se zove `Join()` koja ima tri argumenta:

- Prvi argument  - Tip spajanja; `INNER`, `LEFT`, `OUTER`, `CROSS`, itd.
- Drugi argument - Tabela sa kojom želite da se spojite.
- Treći argument - Uslov spajanja.

### GroupBy i HAVING

`beedb` takođe ima implementaciju `GroupBy` i `Having` metoda.

```go
a, _ := orm.SetTable("userinfo").GroupBy("username").Having("username='astaxie'").FindMap()
```

- `GroupBy()` označava polje koje je namenjeno za grupisanje.
- `Having()` ukazuje na uslove pripadanja grupi.

### Budućnost

Dobio sam mnogo povratnih informacija o `beedb`-u od mnogih ljudi širom sveta i razmišljam o rekonfiguraciji sledećih aspekata:

- Implementiranje dizajna interfejsa sličan onom `database/sql/driver` kako biste olakšali CRUD operacije.

- Implementiranje asocijacije relacionih baza podataka kao što su `jedan prema jednom`, `jedan prema mnogima` i `mnogi prema mnogima`. Evo primera:

```go
type Profile struct {
    Nickname string
    Mobile   string
}

type Userinfo struct {
    Uid         int
    PK_Username string
    Departname  string
    Created     time.Time
    Profile     HasOne
}
```

- Automatski kreirajte tabele i indekse.
- Implementiranje pula konekcija koristeći gorutine.

## 5.6 NoSQL baza podataka

NoSQL baza podataka pruža mehanizam za skladištenje i preuzimanje podataka koji koristi labavije modele konzistentnosti od tipičnih relacionih baza podataka kako bi se postiglo horizontalno skaliranje i veća dostupnost. Neki autori ih nazivaju "Ne samo SQL" kako bi naglasili da neki NoSQL sistemi dozvoljavaju upotrebu jezika za upite sličnih SQL-u.

Kao C jezik 21. veka, Go ima dobru podršku za NoSQL baze podataka, uključujući popularne "Redis", "mongoDB", "Cassandra" i "Membase" NoSQL baze podataka.

### Redis

Redis je sistem za skladištenje ključ-vrednost poput Memcached-a, koji podržava tipove vrednosti `string`, `list`, `set`, `zset` (uređeni skup) i `hash`.

Postoje neki Go drajveri za baze podataka za Redis:

<https://github.com/gomodule/redigo>
<https://github.com/go-redis/redis>
<https://github.com/hoisie/redis>
<https://github.com/alphazero/Go-Redis>
<https://github.com/simonz05/godis>

Da vidimo kako se koristi drajver koji redigo koristi za rad sa bazom podataka:

```go
package main

import (
    "fmt"
    "github.com/gomodule/redigo/redis"
    "os"
    "os/signal"
    "syscall"
    "time"
)
var (
    Pool *redis.Pool
)
func init() {
    redisHost := ":6379"
    Pool = newPool(redisHost)
    close()
}
func newPool(server string) *redis.Pool {
    return &redis.Pool{
        MaxIdle:     3,
        IdleTimeout: 240 * time.Second,
        Dial: func() (redis.Conn, error) {
            c, err := redis.Dial("tcp", server)
            if err != nil {
                return nil, err
            }
            return c, err
        },
        TestOnBorrow: func(c redis.Conn, t time.Time) error {
            _, err := c.Do("PING")
            return err
        },
    }
}
func close() {
    c := make(chan os.Signal, 1)
    signal.Notify(c, os.Interrupt)
    signal.Notify(c, syscall.SIGTERM)
    signal.Notify(c, syscall.SIGKILL)
    go func() {
        <-c
        Pool.Close()
        os.Exit(0)
    }()
}
func Get(key string) ([]byte, error) {
    conn := Pool.Get()
    defer conn.Close()
    var data []byte
    data, err := redis.Bytes(conn.Do("GET", key))
    if err != nil {
        return data, fmt.Errorf("error get key %s: %v", key, err)
    }
    return data, err
}
func main() {
    test, err := Get("test")
    fmt.Println(test, err)
}
```

Napravio sam fork poslednjeg od ovih paketa, ispravio neke greške i koristio ga u svom servisu za kratke URL-ove (2 miliona PV svakog dana).

<https://github.com/astaxie/goredis>

Da vidimo kako da koristimo drajver koji sam podelio da bi radio na bazi podataka:

```go
package main

import (
    "github.com/astaxie/goredis"
    "fmt"
)
func main() {
    var client goredis.Client
    
    // Set the default port in Redis
    client.Addr = "127.0.0.1:6379"
    
    // string manipulation
    client.Set("a", []byte("hello"))
    val, _ := client.Get("a")
    fmt.Println(string(val))
    client.Del("a")
    // list operation
    vals := []string{"a", "b", "c", "d", "e"}
    for _, v := range vals {
        client.Rpush("l", []byte(v))
    }
    dbvals,_ := client.Lrange("l", 0, 4)
    for i, v := range dbvals {
        println(i,":",string(v))
    }
    client.Del("l")
}
```

Možemo videti da je prilično lako koristiti rediS u Go-u i da ima visoke performanse. Njegove klijentske komande su gotovo iste kao i rediS-ove ugrađene komande.

### mongoDB

mongoDB (od „humongous“) je sistem baza podataka otvorenog koda orijentisan na dokumente, koji je razvio i podržava 10gen. Deo je NoSQL porodice sistema baza podataka. Umesto skladištenja podataka u tabelama kao što se to radi u "klasičnoj" relacionoj bazi podataka, MongoDB skladišti strukturirane podatke kao JSON-slične dokumente sa dinamičkim šemama (MongoDB naziva format BSON), što olakšava i ubrzava integraciju podataka u određenim tipovima aplikacija.

![5.1](images/5.6.mongodb.png)  
Slika 5.1 MongoDB u poređenju sa Mysql-om

Najbolji drajver za mongoDB se zove `mgo`, i moguće je da će u budućnosti biti uključen u standardnu biblioteku.

Instaliranje `mgo`:

```go
go get gopkg.in/mgo.v2
```

Evo primera:

```go
package main

import (
    "fmt"
    "gopkg.in/mgo.v2"
    "gopkg.in/mgo.v2/bson"
    "log"
)
type Person struct {
    Name  string
    Phone string
}
func main() {
    session, err := mgo.Dial("server1.example.com,server2.example.com")
    if err != nil {
        panic(err)
    }
    defer session.Close()
    // Optional. Switch the session to a monotonic behavior.
    session.SetMode(mgo.Monotonic, true)
    c := session.DB("test").C("people")
    err = c.Insert(&Person{"Ale", "+55 53 8116 9639"},
        &Person{"Cla", "+55 53 8402 8510"})
    if err != nil {
        log.Fatal(err)
    }
    result := Person{}
    err = c.Find(bson.M{"name": "Ale"}).One(&result)
    if err != nil {
        log.Fatal(err)
    }
    fmt.Println("Phone:", result.Phone)
}
```

Možemo videti da nema velikih razlika kada je u pitanju rad na `mgo` ili `beedb` bazama podataka; obe su zasnovane na strukturama. Ovo je Go način rada.
