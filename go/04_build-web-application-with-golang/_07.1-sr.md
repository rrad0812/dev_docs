# 7.1 XML

[Sadržaj](_00.0-sr.md)

XML je često korišćeni format za komunikaciju podataka u veb servisima. Danas, on preuzima sve važniju ulogu u veb razvoju. U ovom odeljku, predstavićemo vam kako da radite sa XML-om putem Go standardne biblioteke.

Neću pokušavati da vas naučim sintaksi ili konvencijama XML-a. Za to, molim vas pročitajte više dokumentacije o samom XML-u. Fokusiraćemo se samo na to kako kodirati i dekodirati XML datoteke u Go-u.

Pretpostavimo da radite u IT sektoru i da morate da se bavite sledećom XML konfiguracionom datotekom:

```xml
<?xml version="1.0" encoding="utf-8"?>
<servers version="1">
    <server>
        <serverName>Shanghai_VPN</serverName>
        <serverIP>127.0.0.1</serverIP>
    </server>
    <server>
        <serverName>Beijing_VPN</serverName>
        <serverIP>127.0.0.2</serverIP>
    </server>
</servers>
```

Gore navedeni XML dokument sadrži dve vrste informacija o vašem serveru: "serverName" i "serverIP". Koristićemo ovaj dokument u sledećim primerima.

## Raščlanjivanje XML-a

Kako da analiziramo ovaj XML dokument? Možemo koristiti `Unmarshal` funkciju u Go `xml` paketu da bismo to uradili.

```go
func Unmarshal(data []byte, v interface{}) error
```

Parametar `data` prima tok podataka iz XML izvora i v predstavlja strukturu u koju želite da ispišete analizirani XML. To je interfejs, što znači da možete konvertovati XML u bilo koju strukturu koju želite. Ovde ćemo govoriti samo o tome kako konvertovati iz XML-a u `struct` tip, jer dele slične strukture stabla.

Primer koda:

```go
package main

import (
    "encoding/xml"
    "fmt"
    "io/ioutil"
    "os"
)

type Recurlyservers struct {
    XMLName     xml.Name `xml:"servers"`
    Version     string   `xml:"version,attr"`
    Svs         []server `xml:"server"`
    Description string   `xml:",innerxml"`
}

type server struct {
    XMLName    xml.Name `xml:"server"`
    ServerName string   `xml:"serverName"`
    ServerIP   string   `xml:"serverIP"`
}

func main() {
    file, err := os.Open("servers.xml") // For read access.     
    if err != nil {
        fmt.Printf("error: %v", err)
        return
    }
    defer file.Close()
    data, err := ioutil.ReadAll(file)
    if err != nil {
        fmt.Printf("error: %v", err)
        return
    }
    v := Recurlyservers{}
    err = xml.Unmarshal(data, &v)
    if err != nil {
        fmt.Printf("error: %v", err)
        return
    }

    fmt.Println(v)
}
```

XML je zapravo struktura podataka tipa stabla i možemo definisati veoma sličnu strukturu koristeći strukture u programu Go, a zatim je koristiti sa `xml.Unmarshal` za konvertovanje iz XML-a u naš objekat strukture. Primer koda će ispisati sledeći sadržaj:

```xml
{{ servers} 1 [{{ server} Shanghai_VPN 127.0.0.1} {{ server} Beijing_VPN 127.0.0.2}]
<server>
    <serverName>Shanghai_VPN</serverName>
    <serverIP>127.0.0.1</serverIP>
</server>
<server>
    <serverName>Beijing_VPN</serverName>
    <serverIP>127.0.0.2</serverIP>
</server>
}
```

Koristimo `xml.Unmarshal` za parsiranje XML dokumenta na odgovarajući objekat strukture. Trebalo bi da vidite da imamo nešto slično xml:"serverName"u našoj strukturi. Ovo je karakteristika struktura koja se poziva struct tags da pomogne pri refleksiji. Pogledajmo ponovo `Unmarshal` definiciju:

```go
func Unmarshal(data []byte, v interface{}) error
```

Prvi argument je XML tok podataka. Drugi argument je tip skladištenja i podržava tipove struktura, isečaka i stringova. Go-ov XML paket koristi refleksiju za mapiranje podataka, tako da sva polja u "v" treba da se izvezu. Međutim, ovo stvara problem: kako zna koje XML polje odgovara mapiranom polju strukture? Odgovor je da XML parser analizira podatke određenim redosledom. Biblioteka će prvo pokušati da pronađe odgovarajuću oznaku strukture. Ako se podudaranje ne može pronaći, onda pretražuje imena polja strukture. Imajte na umu da su sve oznake, imena polja i XML elementi osetljivi na velika i mala slova, tako da morate da se uverite da postoji jednaka podudarnost da bi mapiranje uspelo.

Go-ov mehanizam refleksije vam omogućava da koristite ove informacije o oznakama za refleksiju XML podataka na objekat strukture. Ako želite da saznate više o refleksiji u Go-u, pročitajte dokumentaciju paketa o oznakama strukture i refleksiji.

Evo nekih pravila pri korišćenju `xml` paketa za parsiranje XML dokumenata u strukture:

- Ako je tip polja `string` ili `[]byte` sa oznakom ",innerxml", Unmarshal će mu dodeliti sirove XML podatke, kao u "Description" u gornjem primeru:

Šangaj_VPN 127.0.0.1 Peking_VPN 127.0.0.2

- Ako se pozove polje `XMLName` tipa `xml.Name`, dobija se ime elementa, kao "servers" u gornjem primeru.

- Ako oznaka polja sadrži odgovarajuće ime elementa, onda dobija i ime elementa, kao "servername" u "servers" u gornjem primeru.

- Ako oznaka polja sadrži ",attr", onda dobija atribut odgovarajućeg elementa, kao version u gornjem primeru.

- Ako oznaka polja sadrži nešto poput "a>b>c", dobija vrednost elementa c čvora b čvora a.

- Ako oznaka polja sadrži "=", onda ono ne dobija ništa.

- Ako oznaka polja sadrži "any", onda dobija sve podređene elemente koji ne odgovaraju ostalim pravilima.

- Ako XML elementi imaju jedan ili više komentara, svi ovi komentari će biti dodati prvom polju koje ima oznaku a koje sadrži "comments". Ovaj tip polja može biti `string` ili `[]byte`. Ako ova vrsta polja ne postoji, svi komentari se odbacuju.

Ova pravila vam govore kako da definišete oznake u strukturama. Kada razumete ova pravila, mapiranje XML-a na strukture biće jednostavno kao i primer koda iznad. Pošto oznake i XML elementi imaju jedan-na-jedan korespondenciju, možemo koristiti i isečke da predstavite više elemenata na istom nivou.

Imajte na umu da sva polja u strukturama treba da budu izvezena (velikim slovima) kako bi se podaci pravilno analizirali.

## Izgradnja XML-a

Šta ako želimo da napravimo XML dokument umesto da ga parsiramo? Kako to da uradimo u Go-u? Nije iznenađujuće da `xml` paket pruža dve funkcije, `Marshal` i `MarshalIndent`, gde druga funkcija automatski uvlači maršalirani XML dokument. Njihova definicija je sledeća:

```go
func Marshal(v interface{}) ([]byte, error)
func MarshalIndent(v interface{}, prefix, indent string) ([]byte, error)
```

Prvi argument u obe ove funkcije je za čuvanje maršaliranog XML toka podataka.

Pogledajmo primer da vidimo kako ovo funkcioniše:

```go
package main

import (
    "encoding/xml"
    "fmt"
    "os"
)

type Servers struct {
    XMLName xml.Name `xml:"servers"`
    Version string   `xml:"version,attr"`
    Svs     []server `xml:"server"`
}

type server struct {
    ServerName string `xml:"serverName"`
    ServerIP   string `xml:"serverIP"`
}

func main() {
    v := &Servers{Version: "1"}
    v.Svs = append(v.Svs, server{"Shanghai_VPN", "127.0.0.1"})
    v.Svs = append(v.Svs, server{"Beijing_VPN", "127.0.0.2"})
    output, err := xml.MarshalIndent(v, "  ", "    ")
    if err != nil {
        fmt.Printf("error: %v\n", err)
    }
    os.Stdout.Write([]byte(xml.Header))

    os.Stdout.Write(output)
}
```

Gore navedeni primer ispisuje sledeće informacije:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<servers version="1">
<server>
    <serverName>Shanghai_VPN</serverName>
    <serverIP>127.0.0.1</serverIP>
</server>
<server>
    <serverName>Beijing_VPN</serverName>
    <serverIP>127.0.0.2</serverIP>
</server>
</servers>
```

Kao što smo prethodno definisali, razlog zašto jesmo `os.Stdout.Write([]byte(xml.Header))` je taj što ni `xml.MarshalIndent`, ni `xml.Marshal` ne prikazuju XML zaglavlja sama po sebi, pa ih moramo eksplicitno ispisati da bismo ispravno proizveli XML dokumente.

Ovde možemo videti da `Marshal` takođe prima parametre tipa `interface{}`. Dakle, koja su pravila prilikom maršaliranja u XML dokument?

- Ako je v string ili isečak, ispisuje sve elemente kao vrednost.
- Ako je v pokazivač, ispisuje sadržaj na koji v pokazuje, ne ispisuje ništa kada je v jednako `nil`.
- Ako je v interfejs, on se bavi i tim interfejsom.
- Ako je v jedan od drugih tipova, ispisuje vrednost tog tipa.

Pa kako `xml.Marshal` određuje ime elemenata? Prati sledeća pravila:

- Ako je v struktura, ona definiše ime u oznaci XMLName.
- Ime polja je XMLName, a tip je xml.Name.
- Oznaka polja u strukturi.
- Ime polja u strukturi.
- Ukucajte ime maršala.

Zatim moramo da shvatimo kako da postavimo oznake da bismo napravili konačni XML dokument.

- XMLName neće biti odštampan.
- Polja koja sadrže oznake "-" neće biti odštampana.
- Ako oznaka sadrži "name,attr", koristi ime kao ime atributa i vrednost polja kao vrednost, kao version u gornjem primeru.
- Ako oznaka sadrži ",attr", koristi naziv polja kao naziv atributa, a vrednost polja kao njegovu vrednost.
- Ako oznaka sadrži ",chardata", ona ispisuje karakterne podatke umesto elementa.
- Ako oznaka sadrži ",innerxml", ona ispisuje sirovu vrednost.
- Ako oznaka sadrži ",comment", ispisuje se kao komentar bez izlaza, tako da ne možete imati "–" u njenoj vrednosti.
- Ako oznaka sadrži "omitempty", ovo polje se izostavlja ako je njegova vrednost nula-vrednost, uključujući false, 0, nil pokazivač ili nil interfejs, nultu dužinu niza, iseečka, mape i stringa.
- Ako oznaka sadrži "a>b>c", ispisuje tri elementa gde a sadrži b, a b sadrži c, kao u sledećem kodu:

FirstName string   `xml:"name>first"`
LastName  string   `xml:"name>last"`

```xml
<name>
<first>Asta</first>
<last>Xie</last>
</name>
```

Možda ste primetili da su strukturne oznake veoma korisne za rad sa XML-om, a isto važi i za ostale formate podataka o kojima ćemo govoriti u narednim odeljcima. Ako i dalje imate problema sa radom sa strukturnim oznakama, verovatno bi trebalo da pročitate više dokumentacije o njima pre nego što pređete na sledeći odeljak.
