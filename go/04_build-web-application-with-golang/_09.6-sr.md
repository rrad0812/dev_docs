
# 9.6 Šifrovanje i dešifrovanje podataka

[Sadržaj](_00.0-sr.md)

U prethodnom odeljku je opisano kako bezbedno čuvati lozinke, ali ponekad može biti potrebno izmeniti neke osetljive šifrovane podatke koji su već sačuvani u našoj bazi podataka. Kada je potrebno dešifrovanje podataka, trebalo bi da koristimo simetrični algoritam za šifrovanje umesto jednosmernih tehnika heširanja koje smo prethodno obradili.

## Napredno šifrovanje i dešifrovanje

Jezik Go podržava simetrične algoritme za šifrovanje u svom `crypto`  paketu. Ne koristite ništa osim AES u GCM režimu ako ne znate šta radite!

- `crypto/aes` paket: AES (Napredni standard šifrovanja), takođe poznat kao "Rijndael" metod šifrovanja, koristi se od strane savezne vlade SAD kao standard za blokovsko šifrovanje.
U sledećem primeru demonstriramo kako se šifruju podaci pomoću AES-a u GCM režimu:

```go
package main

import (
    "crypto/aes"
    "crypto/cipher"
    "crypto/rand"
    "errors"
    "fmt"
    "io"
    "log"
)

func main() {
    text := []byte("My name is Astaxie")
    key := []byte("the-key-has-to-be-32-bytes-long!")

    ciphertext, err := encrypt(text, key)
    if err != nil {
        // TODO: Properly handle error
        log.Fatal(err)
    }
    fmt.Printf("%s => %x\n", text, ciphertext)

    plaintext, err := decrypt(ciphertext, key)
    if err != nil {
        // TODO: Properly handle error
        log.Fatal(err)
    }
    fmt.Printf("%x => %s\n", ciphertext, plaintext)
}

func encrypt(plaintext []byte, key []byte) ([]byte, error) {
    c, err := aes.NewCipher(key)
    if err != nil {
        return nil, err
    }

    gcm, err := cipher.NewGCM(c)
    if err != nil {
        return nil, err
    }

    nonce := make([]byte, gcm.NonceSize())
    if _, err = io.ReadFull(rand.Reader, nonce); err != nil {
        return nil, err
    }

    return gcm.Seal(nonce, nonce, plaintext, nil), nil
}

func decrypt(ciphertext []byte, key []byte) ([]byte, error) {
    c, err := aes.NewCipher(key)
    if err != nil {
        return nil, err
    }

    gcm, err := cipher.NewGCM(c)
    if err != nil {
        return nil, err
    }

    nonceSize := gcm.NonceSize()
    if len(ciphertext) < nonceSize {
        return nil, errors.New("ciphertext too short")
    }

    nonce, ciphertext := ciphertext[:nonceSize], ciphertext[nonceSize:]
    return gcm.Open(nil, nonce, ciphertext, nil)
}
```

Pozivanje gornje funkcije `aes.NewCipher` (čiji parametar ključa `[]byte` mora biti 16, 24 ili 32, što odgovara algoritmima AES-128, AES-192 ili AES-256, respektivno), vraća interfejs `cipher.Block` koji implementira tri funkcije:

```go
type Block interface {
    // BlockSize returns the cipher's block size.
    BlockSize() int

    // Encrypt encrypts the first block in src into dst.
    // Dst and src may point at the same memory.
    Encrypt(dst, src []byte)

    // Decrypt decrypts the first block in src into dst.
    // Dst and src may point at the same memory.
    Decrypt(dst, src []byte)
}
```

Ove tri funkcije implementiraju operacije šifrovanja i dešifrovanja; pogledajte dokumentaciju za Go za detaljnije objašnjenje.

## Rezime

Ovaj odeljak opisuje algoritme za šifrovanje koji se mogu koristiti na različite načine u skladu sa potrebama vaše veb aplikacije za šifrovanje i dešifrovanje. Za aplikacije sa čak i osnovnim bezbednosnim zahtevima preporučuje se korišćenje AES-a u GCM režimu.
