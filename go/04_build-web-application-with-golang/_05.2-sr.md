# 5.2 MySQL

[Sadržaj](_00.0-sr.md)

LAMP stek je poslednjih godina bio veoma popularan na internetu, a M u LAMP-u označava MySQL. MySQL je poznat jer je otvorenog koda i jednostavan za korišćenje. Kao takav, postao je de fakto baza podataka u pozadinskim sistemima mnogih veb-sajtova.

## MySQL drajveri

Postoji nekoliko drajvera koji podržavaju MySQL u Go-u. Neki od njih implementiraju interfejs `database/sql`, dok drugi koriste sopstvene standarde interfejsa.

<https://github.com/go-sql-driver/mysql> podržava `database/sql`, napisan u čistom Go-u.
<https://github.com/ziutek/mymysql> podržava `database/sql` korisnički definisane interfejse, napisane u čistom Go jeziku.

U sledećim primerima ću koristiti prvi drajver (koristim ga i u svojim ličnim projektima), a preporučujem da ga koristite i vi iz sledećih razloga:

- To je novi drajver baze podataka i podržava više funkcija.
- U potpunosti podržava `database/sql` standarde interfejsa.
- Podržava dugotrajne veze koje se održavaju u realnom vremenu sa bezbednošću niti.

## Primer za MySQL

U sledećim odeljcima, koristiću istu strukturu tabele baze podataka za različite baze podataka, a zatim ću kreirati SQL na sledeći način:

```sql
CREATE TABLE `userinfo` (
    `uid` INT(10) NOT NULL AUTO_INCREMENT,
    `username` VARCHAR(64) NULL DEFAULT NULL,
    `department` VARCHAR(64) NULL DEFAULT NULL,
    `created` DATE NULL DEFAULT NULL,
    PRIMARY KEY (`uid`)
);
```

Sledeći primer pokazuje kako se radi sa bazom podataka na osnovu `database/sql` standarda interfejsa.

```go
package main

import (
    _ "github.com/go-sql-driver/mysql"
    "database/sql"
    "fmt"
)

func main() {
    db, err := sql.Open("mysql", "astaxie:astaxie@/test?charset=utf8")
    checkErr(err)

    // insert
    stmt, err := db.Prepare("INSERT userinfo SET username=?,department=?,created=?")
    checkErr(err)

    res, err := stmt.Exec("astaxie", "研发部门", "2012-12-09")
    checkErr(err)

    id, err := res.LastInsertId()
    checkErr(err)
    fmt.Println(id)

// update
    stmt, err = db.Prepare("update userinfo set username=? where uid=?")
    checkErr(err)

    res, err = stmt.Exec("astaxieupdate", id)
    checkErr(err)

    affect, err := res.RowsAffected()
    checkErr(err)
    fmt.Println(affect)

    // query
    rows, err := db.Query("SELECT * FROM userinfo")
    checkErr(err)

    for rows.Next() {
        var uid int
        var username string
        var department string
        var created string
        
        err = rows.Scan(&uid, &username, &department, &created)
        checkErr(err)

        fmt.Println(uid)
        fmt.Println(username)
        fmt.Println(department)
        fmt.Println(created)
    }
    
    rows.Close()

    // delete
    stmt, err = db.Prepare("delete from userinfo where uid=?")
    checkErr(err)

    res, err = stmt.Exec(id)
    checkErr(err)

    affect, err = res.RowsAffected()
    checkErr(err)
    fmt.Println(affect)

    db.Close()
}

func checkErr(err error) {
    if err != nil {
        panic(err)
    }
}
```

Dozvolite mi da ovde objasnim nekoliko važnih funkcija:

- `sql.Open()` otvara registrovani drajver baze podataka. Go-MySQL-Driver je ovde registrovao "mysql" drajver. Drugi argument je DSN (naziv izvora podataka) koji definiše informacije koje se odnose na vezu sa bazom podataka. Podržava sledeće formate:

  - user@unix(/path/to/socket)/dbname?charset=utf8
  - user:password@tcp(localhost:5555)/dbname?charset=utf8
  - user:password@/dbname
  - user:password@tcp([de:ad:be:ef::ca:fe]:80)/dbname

- `db.Prepare()` vraća SQL operaciju koja će biti izvršena. Takođe vraća status izvršenja nakon izvršavanja SQL-a.

- `db.Query()` izvršava SQL i vraća rezultat tipa `Rows`.

- `stmt.Exec()` izvršava SQL koji je pripremljen i sačuvan u Stmt.

Imajte na umu da koristimo format `?` za prosleđivanje argumenata. Ovo je neophodno za sprečavanje SQL injekcija napada.
