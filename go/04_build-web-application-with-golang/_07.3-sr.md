
# 7.3 Regexp

[Sadržaj](_00.0-sr.md)

Regularni izrazi (`Regexp`) su komplikovan, ali moćan alat za uparivanje obrazaca i manipulaciju tekstom. Iako ne funkcioniše tako dobro kao čisto uparivanje teksta, fleksibilniji je. Na osnovu svoje sintakse, možete filtrirati skoro svaku vrstu teksta iz izvornog sadržaja. Ako treba da prikupite podatke u veb razvoju, nije teško koristiti `Regexp` za preuzimanje značajnih podataka.

Go ima `regexp` paket koji pruža zvaničnu podršku za `regexp`. Ako ste već koristili `regexp` u drugim programskim jezicima, trebalo bi da ste upoznati sa njim. Imajte na umu da je Go implementirao RE2 standard, osim za \C. Za više detalja, pratite ovaj link: <http://code.google.com/p/re2/wiki/Syntax>.

Go-ov `strings` paket zapravo može da obavlja mnoge poslove kao što su pretraživanje (`Contains`, `Index`), zamena (`Replace`), parsiranje (`Split`, `Join`) itd., i brži je od Regexp-a. Međutim, sve su to trivijalne operacije. Ako želite da pretražujete string koji nije osetljiv na velika i mala slova, Regexp bi trebalo da bude vaš najbolji izbor. Dakle, ako je `strings` paket dovoljan za vaše potrebe, jednostavno ga koristite jer je jednostavan za korišćenje i čitanje; ako treba da izvršite naprednije operacije, koristite `Regexp`.

Ako se sećate validacije formulara iz prethodnih odeljaka, koristili smo `Regexp` da bismo proverili validnost korisnički unetih informacija. Imajte na umu da su svi znakovi UTF-8. Hajde da saznamo više o Go `regexp` paketu!

## Match

Paket `regexp` ima 3 funkcije za podudaranje: ako se podudara sa šablonom, onda vraća vrednost `true`, u suprotnom vraća vrednost `false`.

```go
func Match(pattern string, b []byte) (matched bool, error error)
func MatchReader(pattern string, r io.RuneReader) (matched bool, error error)
func MatchString(pattern string, s string) (matched bool, error error)
```

Sve 3 funkcije proveravaju da li se `pattern` podudara sa ulaznim izvorom, vraćajući vrednost `true` ako se podudara. Međutim, ako vaš regularni izraz ima sintaksičke greške, vratiće grešku. 3 ulazna izvora ovih funkcija su `slice of byte`, `RuneReader` i `string`.

Evo primera kako da potvrdite IP adresu:

```go
func IsIP(ip string) (b bool) {
    if m, _ := regexp.MatchString("^[0-9]{1,3}\\.[0-9]{1,3}\\.[0-9]{1,3}\\.[0-9]{1,3}$", ip); !m {
        return false
    }
    return true
}
```

Kao što vidite, korišćenje šablona u `regexp` paketu nije toliko drugačije. Evo još jednog primera o proveri da li je korisnički unos validan:

```go
func main() {
    if len(os.Args) == 1 {
        fmt.Println("Usage: regexp [string]")
        os.Exit(1)
    } else if m, _ := regexp.MatchString("^[0-9]+$", os.Args[1]); m {
        fmt.Println("Number")
    } else {
        fmt.Println("Not number")
    }
}
```

U gornjim primerima koristimo `Match(Reader|String` da proverimo da li je sadržaj validan, ali su svi jednostavni za korišćenje.

## Filter

Režim podudaranja može da verifikuje sadržaj, ali ne može da ga iseče, filtrira ili prikuplja podatke iz njega. Ako želite to da uradite, morate da koristite složeni režim `Regexp`-a.

Recimo da treba da napišemo pretraživač. Evo primera kada morate da koristite `Regexp` za filtriranje i isecanje podataka.

```go
package main

import (
    "fmt"
    "io/ioutil"
    "net/http"
    "regexp"
    "strings"
)

func main() {
    resp, err := http.Get("http://www.baidu.com")
    if err != nil {
        fmt.Println("http get error.")
    }
    defer resp.Body.Close()
    
    body, err := ioutil.ReadAll(resp.Body)
    if err != nil {
        fmt.Println("http read error")
        return
    }

    src := string(body)

    // Convert HTML tags to lower case.
    re, _ := regexp.Compile("\\<[\\S\\s]+?\\>")
    src = re.ReplaceAllStringFunc(src, strings.ToLower)

    // Remove STYLE.
    re, _ = regexp.Compile("\\<style[\\S\\s]+?\\</style\\>")
    src = re.ReplaceAllString(src, "")

    // Remove SCRIPT.
    re, _ = regexp.Compile("\\<script[\\S\\s]+?\\</script\\>")
    src = re.ReplaceAllString(src, "")

    // Remove all HTML code in angle brackets, and replace with newline.
    re, _ = regexp.Compile("\\<[\\S\\s]+?\\>")
    src = re.ReplaceAllString(src, "\n")

    // Remove continuous newline.
    re, _ = regexp.Compile("\\s{2,}")
    src = re.ReplaceAllString(src, "\n")

    fmt.Println(strings.TrimSpace(src))
}
```

U ovom primeru, koristimo kompajliranje kao prvi korak za složeni režim. Kod proverava da li je vaša regularna sintaksa ispravna, a zatim vraća vrednost `Regexp` za parsiranje sadržaja u drugim operacijama.

Evo nekih funkcija za analizu vaše Regexp sintakse:

```go
func Compile(expr string) (*Regexp, error)
func CompilePOSIX(expr string) (*Regexp, error)
func MustCompile(str string) *Regexp
func MustCompilePOSIX(str string) *Regexp
```

Razlika između `ComplePOSIX` i `Compile` je u tome što prvi mora da koristi POSIX sintaksu koja je najduža pretraga krajnje levo, a drugi je samo pretraga krajnje levo. Na primer, za `Regexp` `[a-z]`{2,4}` i content "aa09aaa88aaaa", `CompilePOSIX` vraća se aaaaa dok `Compile` vraća prefiks `aa`.`Must` znači paniku kada sintaksa `Regexp` nije ispravna, u suprotnom vraća grešku.

Sada kada znamo kako da kreiramo novi Regexp, hajde da vidimo kako nam metode koje pruža ova struktura mogu pomoći u radu sa sadržajem:

```go
func (re *Regexp) Find(b []byte) []byte
func (re *Regexp) FindAll(b []byte, n int) [][]byte
func (re *Regexp) FindAllIndex(b []byte, n int) [][]int
func (re *Regexp) FindAllString(s string, n int) []string
func (re *Regexp) FindAllStringIndex(s string, n int) [][]int
func (re *Regexp) FindAllStringSubmatch(s string, n int) [][]string
func (re *Regexp) FindAllStringSubmatchIndex(s string, n int) [][]int
func (re *Regexp) FindAllSubmatch(b []byte, n int) [][][]byte
func (re *Regexp) FindAllSubmatchIndex(b []byte, n int) [][]int
func (re *Regexp) FindIndex(b []byte) (loc []int)
func (re *Regexp) FindReaderIndex(r io.RuneReader) (loc []int)
func (re *Regexp) FindReaderSubmatchIndex(r io.RuneReader) []int
func (re *Regexp) FindString(s string) string
func (re *Regexp) FindStringIndex(s string) (loc []int)
func (re *Regexp) FindStringSubmatch(s string) []string
func (re *Regexp) FindStringSubmatchIndex(s string) []int
func (re *Regexp) FindSubmatch(b []byte) [][]byte
func (re *Regexp) FindSubmatchIndex(b []byte) []int
```

Ovih 18 metoda uključuje identične funkcije za različite ulazne izvore (bajtni isečak, string i io.RuneReader), tako da možemo zaista pojednostaviti ovu listu ignorišući ulazne izvore na sledeći način:

```go
func (re *Regexp) Find(b []byte) []byte
func (re *Regexp) FindAll(b []byte, n int) [][]byte
func (re *Regexp) FindAllIndex(b []byte, n int) [][]int
func (re *Regexp) FindAllSubmatch(b []byte, n int) [][][]byte
func (re *Regexp) FindAllSubmatchIndex(b []byte, n int) [][]int
func (re *Regexp) FindIndex(b []byte) (loc []int)
func (re *Regexp) FindSubmatch(b []byte) [][]byte
func (re *Regexp) FindSubmatchIndex(b []byte) []int
```

Primer koda:

```go
package main

import (
    "fmt"
    "regexp"
)

func main() {
    a := "I am learning Go language"

    re, _ := regexp.Compile("[a-z]{2,4}")

    // Find the first match.
    one := re.Find([]byte(a))
    fmt.Println("Find:", string(one))

    // Find all matches and save to a slice, n less than 0 means return all matches, indicates length of slice if it's greater than 0.
    all := re.FindAll([]byte(a), -1)
    fmt.Println("FindAll", all)

    // Find index of first match, start and end position.
    index := re.FindIndex([]byte(a))
    fmt.Println("FindIndex", index)

    // Find index of all matches, the n does same job as above.
    allindex := re.FindAllIndex([]byte(a), -1)
    fmt.Println("FindAllIndex", allindex)

    re2, _ := regexp.Compile("am(.*)lang(.*)")

    // Find first submatch and return array, the first element contains all elements, the second element contains the result of first (), the third element contains the result of second ().
    // Output: 
    // the first element: "am learning Go language"
    // the second element: " learning Go ", notice spaces will be outputed as well.
    // the third element: "uage"
    submatch := re2.FindSubmatch([]byte(a))
    fmt.Println("FindSubmatch", submatch)
    for _, v := range submatch {
        fmt.Println(string(v))
    }

    // Same as FindIndex().
    submatchindex := re2.FindSubmatchIndex([]byte(a))
    fmt.Println(submatchindex)

    // FindAllSubmatch, find all submatches.
    submatchall := re2.FindAllSubmatch([]byte(a), -1)
    fmt.Println(submatchall)

    // FindAllSubmatchIndex,find index of all submatches.
    submatchallindex := re2.FindAllSubmatchIndex([]byte(a), -1)
    fmt.Println(submatchallindex)
}
```

Kao što smo već pomenuli, Regexp takođe ima 3 metode za uparivanje. One rade potpuno istu stvar kao i izvezene funkcije. U stvari, te izvezene funkcije zapravo pozivaju ove metode "ispod haube":

```go
func (re *Regexp) Match(b []byte) bool
func (re *Regexp) MatchReader(r io.RuneReader) bool
func (re *Regexp) MatchString(s string) bool

Zatim, da vidimo kako da zamenimo stringove koristeći Regexp:

func (re *Regexp) ReplaceAll(src, repl []byte) []byte
func (re *Regexp) ReplaceAllFunc(src []byte, repl func([]byte) []byte) []byte
func (re *Regexp) ReplaceAllLiteral(src, repl []byte) []byte
func (re *Regexp) ReplaceAllLiteralString(src, repl string) string
func (re *Regexp) ReplaceAllString(src, repl string) string
func (re *Regexp) ReplaceAllStringFunc(src string, repl func(string) string) string
```

Ovo se koristi u primeru sa pretraživanjem, tako da ovde nećemo dalje objašnjavati.

Hajde da pogledamo definiciju `Expand`:

```go
func (re *Regexp) Expand(dst []byte, template []byte, src []byte, match []int) []byte
func (re *Regexp) ExpandString(dst []byte, template string, src string, match []int) []byte
```

Pa kako da koristimo `Expand`?

```go
func main() {
    src := []byte(`
        call hello alice
        hello bob
        call hello eve
    `)
    pat := regexp.MustCompile(`(?m)(call)\s+(?P<cmd>\w+)\s+(?P<arg>.+)\s*$`)
    res := []byte{}
    for _, s := range pat.FindAllSubmatchIndex(src, -1) {
        res = pat.Expand(res, []byte("$cmd('$arg')\n"), src, s)
    }
    fmt.Println(string(res))
}
```

U ovom trenutku, naučili ste ceo `regexp` paket Go jezika. Nadam se da možete razumeti više proučavajući primere ključnih metoda, kako biste mogli sami da uradite nešto zanimljivo.
